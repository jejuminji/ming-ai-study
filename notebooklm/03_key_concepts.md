# 초보자가 먼저 잡아야 할 핵심 개념

## 이 문서의 목적
원본 문서를 깊게 읽기 전에, 꼭 먼저 알아야 할 핵심 개념을 쉬운 말로 정리합니다.

## 추천 독자 수준
beginner

## 아주 짧은 요약
AI를 처음 공부할 때는 모든 용어를 다 외우는 것보다, 큰 개념 몇 개의 관계를 먼저 이해하는 것이 더 중요합니다.

## 1. AI
쉬운 말: 기계가 사람처럼 판단하거나 생성하도록 만드는 넓은 분야입니다.

조금 더 기술적으로: 머신러닝, 딥러닝, 강화학습, 생성형 AI 같은 여러 하위 분야를 포함하는 큰 범주입니다.

왜 중요하나: 이 레포 전체가 결국 AI를 공부하는 길잡이이기 때문입니다.

## 2. 머신러닝
쉬운 말: 사람이 규칙을 다 써 주는 대신, 데이터를 보고 패턴을 배우게 하는 방법입니다.

예시:
- 스팸 메일 구분
- 집값 예측
- 고객 묶기

왜 중요하나: 딥러닝과 LLM도 넓게 보면 머신러닝의 흐름 위에 있습니다.

## 3. 딥러닝
쉬운 말: 신경망을 여러 층으로 쌓아 더 복잡한 패턴을 배우는 방법입니다.

중요한 키워드:
- 신경망
- weight
- bias
- 활성화 함수
- 역전파

왜 중요하나: LLM도 결국 딥러닝 위에 만들어집니다.

## 4. LLM
쉬운 말: 아주 많은 텍스트를 학습한 언어 모델입니다.

더 쉽게 말하면: "매우 똑똑한 자동완성"처럼 시작해서, 문맥을 보고 다음 말을 잘 이어 가는 모델입니다.

관련 용어:
- 토큰: 모델이 읽는 작은 단위
- 임베딩: 의미를 숫자로 바꾼 것
- 어텐션: 어떤 정보를 더 중요하게 볼지 정하는 방식

왜 중요하나: 요즘 생성형 AI의 중심 개념이기 때문입니다.

## 5. 프롬프팅
쉬운 말: 모델에게 더 잘 질문하거나 지시하는 방법입니다.

왜 중요하나: 같은 모델이어도 입력을 어떻게 주느냐에 따라 결과가 많이 달라질 수 있기 때문입니다.

## 6. RAG
쉬운 말: 모델이 자기 머리만 쓰는 게 아니라, 바깥 문서를 찾아보고 답하게 만드는 방식입니다.

비유: 시험 볼 때 기억만으로 답하는 게 아니라, 옆에 참고자료를 두고 보는 느낌입니다.

왜 중요하나: 최신 정보나 내부 문서를 반영할 때 자주 쓰입니다.

## 7. 파인튜닝
쉬운 말: 모델을 어떤 목적에 맞게 다시 조금 더 학습시키는 것입니다.

RAG와 차이:
- 파인튜닝: 모델 자체의 습관이나 행동을 조정
- RAG: 바깥 자료를 가져와 참고하게 함

왜 중요하나: 둘을 헷갈리면 LLM 활용 전략이 흐려지기 쉽기 때문입니다.

## 8. 에이전트
쉬운 말: 모델이 답만 하는 것을 넘어, 도구를 쓰고, 계획을 세우고, 여러 단계를 처리하게 만든 시스템입니다.

예시:
- 검색하기
- 계산하기
- 코드 실행하기
- 상태를 기억하기

왜 중요하나: 요즘 "AI가 일을 한다"는 말은 종종 에이전트형 구조를 뜻하기 때문입니다.

## 9. 평가
쉬운 말: 모델이나 시스템이 잘 되고 있는지 확인하는 과정입니다.

자주 나오는 지표:
- 정확도
- 정밀도
- 재현율
- F1

왜 중요하나: 잘 만들었다고 느끼는 것과 실제로 잘 되는 것은 다를 수 있기 때문입니다.

## 10. 시스템 설계
쉬운 말: 좋은 모델만 고르는 것이 아니라, 입력 처리, 검색, 메모리, 가드레일, 평가까지 함께 설계하는 일입니다.

왜 중요하나: 실제 서비스는 모델 하나만으로 완성되지 않기 때문입니다.

## 이 레포에서 특히 중요한 연결 관계

| 먼저 이해할 것 | 그 다음 이해할 것 |
| --- | --- |
| AI | ML, DL |
| DL | 신경망, 학습 알고리즘 |
| 신경망 기초 | LLM |
| LLM | 프롬프팅, RAG, 에이전트 |
| 모델 이해 | 평가, 시스템 설계 |

## 헷갈리기 쉬운 쌍

### AI vs LLM
AI는 큰 분야이고, LLM은 그 안의 일부입니다.

### 딥러닝 vs LLM
딥러닝은 방법이고, LLM은 그 방법으로 만든 큰 언어 모델입니다.

### 프롬프팅 vs RAG
프롬프팅은 입력을 더 잘 쓰는 것이고, RAG는 바깥 자료를 붙이는 것입니다.

### RAG vs 파인튜닝
RAG는 참고자료를 붙이는 방식이고, 파인튜닝은 모델 자체를 조정하는 방식입니다.

### 모델 vs 시스템
모델은 부품 하나에 가깝고, 시스템은 여러 부품이 함께 움직이는 전체 구조입니다.
