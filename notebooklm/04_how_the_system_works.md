# 이 레포는 전체적으로 어떻게 작동하는가

## 이 문서의 목적
이 저장소를 초보자 관점에서 "입력 → 처리 → 출력" 흐름으로 설명합니다.

## 추천 독자 수준
beginner

## 아주 짧은 요약
이 레포는 하나의 앱이 돌아가는 시스템이라기보다, 문서와 예제가 서로 연결되어 학습이 진행되는 구조입니다.

## 먼저 가장 중요한 한 줄
이 레포의 핵심은 "코드를 실행하는 것"보다 "개념을 연결해서 이해하는 것"입니다.

## 이 레포의 전체 흐름을 아주 쉽게 보기

### 입력
여기서 입력은 보통 두 가지입니다.
- 공부하려는 사람의 질문
- 처음 보는 개념에 대한 혼란

예를 들면 이런 질문이 입력입니다.
- AI와 ML은 뭐가 다르지?
- LLM은 어떻게 동작하지?
- RAG와 파인튜닝은 뭐가 다르지?
- 예제 코드는 어디서 보면 되지?

### 처리
이 레포는 그 질문을 여러 종류의 자료로 나눠 처리합니다.

| 자료 위치 | 무슨 역할을 하나 |
| --- | --- |
| `docs/` | 개념을 설명한다 |
| `examples/` | OpenAI API 예제를 보여 준다 |
| `ai-python-study/` | AI Python 코드에서 자주 보는 함수를 쉽게 해설한다 |
| `docs/12_projects_i_worked_on.md` | 실제 프로젝트 경험과 개념을 연결한다 |
| `docs/10_glossary.md`, `docs/AI_용어집_노션용.md` | 용어를 풀어 준다 |

### 출력
이 레포를 통해 기대하는 출력은 다음과 같습니다.
- 개념 이해
- 용어 정리
- 예제 코드 감각
- 질문에 대한 정리된 답
- NotebookLM용 요약, 슬라이드, 발표 자료의 재료

## 단계별로 보면 이렇게 이어진다

### 1단계: 전체 지도 보기
먼저 `README.md`와 `docs/00_AI_roadmap.md`가 "이 레포 전체가 무엇을 다루는지"를 보여 줍니다.

### 2단계: 기초 개념 잡기
`docs/01~04`는 AI, 머신러닝, 딥러닝, 신경망 기초를 설명합니다.  
이 단계는 기초 뼈대를 만드는 구간입니다.

### 3단계: 신경망 내부 원리 보강
`docs/13~15`는 경사하강법, SGD, Adam, 분류, MLP, Backpropagation, CNN/RNN/GAN 같은 내용을 더 자세히 설명합니다.

### 4단계: LLM과 생성형 AI 보기
`docs/05~07`은 트랜스포머, LLM, 프롬프팅, RAG, 에이전트를 설명합니다.

### 5단계: 실제 운영 관점 보기
`docs/08~09`는 평가와 시스템 설계를 다룹니다.  
즉, "모델이 있으면 끝"이 아니라 "서비스로 만들려면 무엇이 더 필요한가"를 봅니다.

### 6단계: 용어, 계획, 프로젝트로 연결하기
`docs/10~12`는 용어, 학습 계획, 프로젝트 경험을 정리해 줍니다.

### 7단계: 예제 코드로 감각 잡기
`examples/`는 OpenAI API 예시를, `ai-python-study/`는 Python 함수 읽기 연습을 돕습니다.

## 누가 누구와 연결되는가

### `README.md` → `docs/`
`README.md`는 입구이고, `docs/`는 본문입니다.

### `docs/` → `examples/`
개념을 읽은 뒤, 실제 API 코드 흐름을 보고 싶을 때 `examples/`로 넘어갑니다.

### `docs/` → `ai-python-study/`
개념은 알겠는데 Python 함수가 아직 낯설다면 `ai-python-study/`로 넘어갑니다.

### `docs/12_projects_i_worked_on.md` → 다른 문서들
프로젝트 문서는 개념들이 실제로 어디에 쓰였는지 보여 주는 연결 고리입니다.

## 이 레포를 시스템이라고 부를 때 주의할 점
- 이 저장소에는 하나의 통합 실행 앱 구조가 문서상 명확히 제시되어 있지 않습니다.
- 그래서 여기서 `system`은 주로 "학습 구조", "개념 연결", "자료 흐름"을 뜻합니다.
- 하나의 서버, 하나의 API, 하나의 배포 구조로 이해하면 오해할 수 있습니다.

이 부분은 `문서 근거 부족`이 아니라, 레포의 성격 자체가 문서형 저장소이기 때문입니다.

## 쉬운 비유
이 레포는 공장 자동화 시스템보다, 잘 정리된 도서관에 가깝습니다.

- `README.md`: 안내 데스크
- `docs/`: 주제별 책장
- `examples/`: 실습 코너
- `ai-python-study/`: 풀이 노트 코너
- `notebooklm/`: 초보자용 정리본 코너
