# 초보자용 쉬운 용어집

## 이 문서의 목적
이 레포를 읽다가 자주 만나는 용어를 아주 쉬운 말로 짧게 설명합니다.

## 추천 독자 수준
beginner

## 아주 짧은 요약
모르는 용어가 나오면 이 문서에서 먼저 짧게 뜻을 잡고, 다시 본문으로 돌아가면 됩니다.

## 용어 정리

| 용어 | 쉬운 설명 |
| --- | --- |
| AI | 기계가 사람처럼 판단하거나 생성하게 만드는 넓은 분야 |
| ML | 데이터를 보고 배우는 방법 |
| DL | 신경망을 깊게 쌓아 배우는 방법 |
| LLM | 아주 많은 텍스트를 학습한 언어 모델 |
| 토큰 | 모델이 읽는 작은 텍스트 단위 |
| 임베딩 | 의미를 숫자 벡터로 바꾼 것 |
| 어텐션 | 어떤 정보를 더 중요하게 볼지 정하는 방식 |
| 컨텍스트 윈도우 | 모델이 한 번에 참고할 수 있는 입력 범위 |
| 추론 | 학습된 모델이 실제 답을 만드는 과정 |
| 프롬프트 | 모델에게 주는 질문이나 지시 |
| 프롬프팅 | 프롬프트를 더 잘 만드는 방법 |
| RAG | 외부 문서를 찾아서 답변에 붙이는 방식 |
| 파인튜닝 | 모델을 목적에 맞게 다시 조금 학습시키는 것 |
| LoRA | 적은 부분만 학습하는 가벼운 파인튜닝 방식 |
| 에이전트 | 모델이 도구를 쓰고 계획하며 여러 단계를 수행하는 시스템 |
| 툴 사용 | 모델이 검색, 계산, API 호출 등을 하는 것 |
| 메모리 | 이전 대화나 상태를 기억하는 구조 |
| 평가 | 모델이나 시스템이 잘 되는지 확인하는 일 |
| 정확도 | 전체 중 맞춘 비율 |
| 정밀도 | 맞다고 한 것 중 실제로 맞은 비율 |
| 재현율 | 실제 정답 중 찾아낸 비율 |
| F1 | 정밀도와 재현율의 균형 지표 |
| Guardrail | 위험한 출력이나 정책 위반을 막는 장치 |
| MLE | 데이터를 가장 그럴듯하게 설명하는 파라미터를 찾는 생각 |
| GD | 손실이 줄어드는 방향으로 조금씩 움직이는 기본 최적화 생각 |
| SGD | 일부 데이터만 보고 자주 업데이트하는 방식 |
| Adam | 딥러닝에서 자주 쓰는 최적화기 |
| PPO | 강화학습에서 자주 쓰이는 정책 학습 알고리즘 |
| CQL | 오프라인 데이터로 강화학습할 때 쓰는 보수적 알고리즘 |
| CNN | 이미지 같은 공간 구조 데이터를 잘 다루는 신경망 |
| RNN | 순서가 중요한 데이터를 다루는 신경망 |
| GAN | 생성자와 판별자가 경쟁하며 학습하는 생성 구조 |
| Backpropagation | 최종 오류를 뒤로 보내며 각 가중치를 업데이트하게 하는 과정 |
| Sigmoid | 값을 0~1 사이로 바꾸는 함수로, 이진 분류에서 자주 보임 |
| Softmax | 여러 점수를 확률 분포처럼 바꾸는 함수 |

## 용어를 볼 때의 팁
- 처음에는 뜻을 완벽히 외우지 않아도 됩니다.
- "이 용어가 무엇을 해결하려는가"만 먼저 이해해도 충분합니다.
- 비슷한 용어는 비교해서 보는 편이 훨씬 쉽습니다.

## 자주 헷갈리는 묶음

### AI / ML / DL / LLM
큰 순서대로 보면 `AI > ML > DL > LLM`에 가깝습니다.

### 프롬프팅 / RAG / 파인튜닝 / 에이전트
- 프롬프팅: 입력 설계
- RAG: 외부 지식 연결
- 파인튜닝: 모델 조정
- 에이전트: 도구 사용 시스템

### GD / SGD / Adam
- GD: 전체적인 기본 아이디어
- SGD: 일부 데이터만 보고 자주 업데이트
- Adam: 더 똑똑하게 업데이트하려는 최적화기
