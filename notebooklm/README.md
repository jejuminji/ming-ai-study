# NotebookLM용 초보자 학습 팩 안내

## 이 문서의 목적
왜 `notebooklm/` 폴더를 따로 만들었는지 설명하고, 이 문서들을 NotebookLM에 어떻게 올리고 활용하면 좋은지 안내합니다.

## 추천 독자 수준
beginner

## 아주 짧은 요약
이 폴더는 원본 `docs/`를 초보자에게 더 쉽게 설명하기 위해 다시 쓴 NotebookLM 업로드용 문서 묶음입니다.

## 왜 이 폴더를 따로 만들었는가
원본 `docs/`는 이미 학습 친화적으로 쓰여 있지만, 초보자 기준에서는 여전히 정보량이 많을 수 있습니다.  
특히 NotebookLM에 올려 질문할 때는, 한 문서가 너무 넓거나 용어가 빠르게 나오면 답변이 조금 딱딱해질 수 있습니다.

그래서 `notebooklm/` 폴더는 아래 원칙으로 다시 구성했습니다.
- 더 쉬운 말부터 설명
- 문서 하나하나가 독립적으로 읽히게 구성
- 구조, 개념, 흐름, 질문을 분리
- NotebookLM이 요약, Q&A, 슬라이드, 발표 개요를 만들기 쉽게 정리

## 이 폴더가 원본 문서와 다른 점
- 원본 문서를 그대로 복사하지 않았습니다.
- 초보자가 헷갈리는 순서를 다시 정리했습니다.
- 어려운 용어는 먼저 쉬운 말로 풀었습니다.
- 실행 정보가 부족한 부분은 `문서 근거 부족`이라고 분명히 적었습니다.

## NotebookLM에 올리는 방법
1. `notebooklm/` 폴더의 Markdown 파일들을 준비합니다.
2. NotebookLM에서 새 노트를 만듭니다.
3. 이 폴더의 `.md` 파일들을 업로드합니다.
4. 먼저 `00_index.md`와 `01_this_repo_in_plain_language.md`를 기준으로 큰 그림 질문을 합니다.
5. 그 다음 개념, 구조, 워크플로우, 발표용 질문으로 확장합니다.

### 구조를 먼저 잡고 싶은 순서
`00` → `02` → `04` → `06` → `05`

### 발표 자료 만들기 좋은 순서
`01` → `03` → `04` → `08` → `10`

## NotebookLM에 던지기 좋은 예시 프롬프트
- "이 레포를 완전 처음 보는 사람 기준으로 설명해줘."
- "핵심 개념만 먼저 쉽게 설명해줘."
- "전체 동작 흐름을 단계별로 설명해줘."
- "가장 중요한 파일과 폴더만 골라서 알려줘."
- "이 내용을 바탕으로 초보자용 5장 슬라이드를 만들어줘."
- "이 프로젝트를 입문자 발표용으로 설명하는 비디오 개요를 만들어줘."

## 이런 식으로 활용하면 좋다

### 1. 처음 공부할 때
`01`, `03`, `08` 중심으로 질문합니다.

### 2. 문서 구조가 헷갈릴 때
`02`, `04`, `06` 중심으로 질문합니다.

### 3. 복습할 때
`09`, `10` 중심으로 질문합니다.

### 4. 발표 준비할 때
`01`, `04`, `08`, `10`을 바탕으로 슬라이드 초안을 만듭니다.

## 주의할 점
- NotebookLM 답변도 업로드한 문서 범위 안에서 이해해야 합니다.
- 이 레포는 실행형 앱보다 문서형 학습 레포에 가깝습니다.
- 실행 절차가 완전하지 않은 부분은 원래 문서에도 근거가 부족할 수 있습니다.

## 핵심 포인트
- `notebooklm/`은 초보자용 질문 응답과 요약에 맞춰 다시 쓴 문서 묶음입니다.
- 원본 `docs/`보다 더 쉬운 설명과 더 작은 단위의 문서 구조를 목표로 했습니다.
- 이 폴더만 올려도 NotebookLM에서 초보자 학습, 요약, 슬라이드 생성에 충분히 활용할 수 있습니다.
