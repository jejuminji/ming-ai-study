# 이 레포를 어떻게 읽고, 써 보고, 활용할까

## 이 문서의 목적
이 저장소를 실제로 어떻게 사용하면 좋은지, 문서 읽기와 예제 실행 관점에서 정리합니다.

## 추천 독자 수준
beginner

## 아주 짧은 요약
이 레포는 문서만 읽어도 공부가 가능하고, 원하면 예제 코드도 일부 실행해 볼 수 있습니다.

## 먼저 중요한 사실
이 레포는 하나의 통합 애플리케이션 실행 레포가 아닙니다.  
즉, `npm start`나 `python app.py`처럼 "전체를 한 번에 실행하는 방식"은 `문서 근거 부족`입니다.

대신 다음 두 가지는 분명합니다.
- 문서 중심으로 공부할 수 있다
- 일부 Python 예제는 따로 실행해 볼 수 있다

## 필수 준비물

### 문서만 읽을 때
- Markdown 파일을 열 수 있는 편집기
- 폴더 구조를 볼 수 있는 파일 탐색기 또는 IDE

### 예제 코드를 실행할 때
- Python 3.10+
- `openai` 패키지
- `OPENAI_API_KEY` 환경 변수

이 부분은 `examples/README.md`에 문서 근거가 있습니다.

### `ai-python-study/` 코드를 볼 때
- Python 기본 실행 환경

추정:
- 이 폴더는 대부분 표준 라이브러리 중심 학습용 코드라서, 별도 복잡한 설치는 크게 필요 없어 보입니다.

## 가장 쉬운 사용 방법: 문서만 읽기
1. `README.md`를 연다.
2. `docs/00_AI_roadmap.md`를 읽는다.
3. `docs/01`, `03`, `04` 순서로 기초를 잡는다.
4. LLM이 궁금하면 `docs/05`, `06`, `07`로 넘어간다.
5. 용어가 막히면 `docs/10_glossary.md`를 본다.

## 예제 코드 사용 방법

### `examples/` 폴더
이 폴더는 OpenAI API 예시를 담고 있습니다.

추천 순서:
1. `기초 텍스트 응답-01.py`
2. `구조화된 출력-02.py`
3. `이미지 이해-03.py`
4. `함수 호출-04.py`

기대 결과:
- API 호출 흐름이 어떻게 생겼는지 보인다
- 코드 주석을 따라가며 각 줄 역할을 이해할 수 있다

자주 막히는 부분:
- API 키가 없으면 실행이 안 된다
- `openai` 패키지가 없으면 import 에러가 날 수 있다

## `ai-python-study/` 폴더 사용 방법
이 폴더는 실행보다 "읽기"가 더 중요합니다.

추천 순서:
1. `README.md`
2. `01_math_and_data_helpers.py`
3. `02_training_and_metrics_helpers.py`
4. `03_nlp_and_rag_helpers.py`
5. `04_llm_prompt_helpers.py`

기대 결과:
- AI 코드에서 자주 나오는 함수 구조에 익숙해진다
- 입력, 처리, 출력 구조를 설명할 수 있게 된다

자주 막히는 부분:
- 실무 코드와 똑같을 것이라고 기대하면 안 된다
- 이 코드는 학습용 단순화가 들어가 있다

## NotebookLM에 올려서 사용하는 방법
1. `notebooklm/README.md`를 먼저 읽는다.
2. `notebooklm/` 폴더 문서를 NotebookLM에 업로드한다.
3. 쉬운 질문부터 던진다.
4. 그다음 슬라이드, 발표 개요, 요약 요청으로 확장한다.

기대 결과:
- 초보자용 설명
- 질문 응답
- 복습 자료
- 발표 초안

## 기대 결과를 현실적으로 보기

### 확실히 기대할 수 있는 것
- 문서 기반 학습
- 개념 연결 이해
- 예제 코드 흐름 읽기
- NotebookLM 기반 요약/질문 활용

### 기대하면 안 되는 것
- 이 레포 하나만으로 완전한 배포형 제품 실행
- 하나의 통합 서비스 서버 실행
- 모든 프로젝트 산출물의 완전한 재현

이 부분은 `문서 근거 부족` 또는 레포 성격상 해당하지 않는 부분입니다.
