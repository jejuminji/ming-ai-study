# 트랜스포머와 LLM

## 이 문서에서 배우는 것
- 트랜스포머와 LLM의 관계를 이해한다.
- 토큰, 임베딩, 어텐션, 컨텍스트 윈도우, 추론, 파인튜닝이 어떻게 이어지는지 본다.
- "챗봇처럼 보이는 것"과 "언어 모델의 구조"를 구분한다.
- LLM이 잘하는 일과 한계를 같이 본다.

## 왜 중요한가
요즘 AI 실무의 많은 논의는 LLM 중심으로 이뤄진다. 하지만 LLM을 단순히 "말 잘하는 모델"로만 이해하면 한계와 가능성을 모두 놓치기 쉽다. 구조를 알면 프롬프트, RAG, 파인튜닝의 차이도 자연스럽게 이해된다.

예를 들어 "왜 긴 문서를 다 넣었는데도 답이 이상하지?", "왜 최신 정보를 모를까?", "왜 비용이 많이 나올까?" 같은 질문은 모두 LLM의 구조적 특성과 연결된다.

## 핵심 개념 설명
LLM은 대규모 텍스트 데이터를 학습해 다음 토큰을 예측하는 언어 모델이다. 입력 문장은 토큰으로 쪼개지고, 각 토큰은 임베딩이라는 벡터 표현으로 바뀐다. 트랜스포머는 어텐션 메커니즘을 통해 문맥 안에서 어떤 정보가 중요한지 계산한다. 이 구조 덕분에 긴 문맥의 관계를 비교적 잘 다룰 수 있다.

연결 흐름은 다음과 같다.
1. 텍스트를 토큰으로 분해
2. 토큰을 임베딩으로 변환
3. 어텐션으로 문맥 관계 계산
4. 다음 토큰을 예측하며 추론
5. 필요하면 추가 학습으로 파인튜닝

핵심 용어:

| 용어 | 쉬운 설명 | 실무 포인트 |
| --- | --- | --- |
| 토큰 | 모델이 읽는 최소 단위 | 비용, 길이 제한과 연결 |
| 임베딩 | 의미를 담은 숫자 표현 | 검색, 유사도 계산과 연결 |
| 어텐션 | 무엇을 더 볼지 결정 | 문맥 이해의 핵심 |
| 컨텍스트 윈도우 | 한 번에 볼 수 있는 범위 | 긴 문서 처리 한계와 연결 |
| 추론 | 학습된 모델이 답 생성 | 속도, 비용, 품질 문제 |
| 파인튜닝 | 특정 목적에 맞게 추가 학습 | 행동/스타일/도메인 적응 |

## 작은 입력 예시
문장 `"오늘 비 와?"`를 모델이 처리한다고 생각해 보자.
1. 문장을 토큰으로 나눈다.
2. 각 토큰을 숫자 벡터로 바꾼다.
3. "오늘", "비", "와?" 사이의 관계를 어텐션으로 본다.
4. 다음에 올 답변 토큰을 하나씩 생성한다.

즉, 모델은 문장을 "의미 있는 벡터들의 관계"로 본다. 사람이 읽는 방식과 완전히 같지는 않지만, 결과적으로는 문맥을 반영한 답을 생성하게 된다.

## 트랜스포머와 챗봇은 같은가
같지 않다.
- 트랜스포머: 모델 구조
- LLM: 그 구조를 대규모 언어 데이터로 학습한 모델
- 챗봇: 그 모델을 대화 UI와 시스템 로직으로 감싼 제품 형태

그래서 챗봇 품질은 모델만이 아니라 시스템 프롬프트, RAG, 메모리, 안전장치, UI 설계에도 영향을 받는다.

## 예시 / 비유
LLM을 "매우 큰 자동완성 시스템"이라고 생각하면 출발점으로 좋다. 다만 단순 문자 자동완성이 아니라, 토큰 의미와 문맥 관계를 벡터 공간에서 계산해 이어 가는 고도화된 자동완성이다.

또 다른 비유로 보면:
- 토큰은 문장의 조각
- 임베딩은 그 조각의 좌표
- 어텐션은 지금 어떤 조각을 더 중요하게 볼지 정하는 렌즈

## 실무에서 자주 부딪히는 한계
- LLM은 데이터베이스가 아니어서 최신 사실을 항상 정확히 기억하지 않는다.
- 컨텍스트가 길어도 중요한 정보를 반드시 잘 활용하는 것은 아니다.
- 입력 토큰이 많아질수록 비용과 지연 시간이 커질 수 있다.
- 추론은 확률적 생성이라서 같은 질문에도 답이 조금씩 달라질 수 있다.

이 한계 때문에 RAG, 평가, 가드레일, 시스템 설계가 같이 필요해진다.

## 한눈에 정리
- 트랜스포머는 구조, LLM은 그 구조를 활용한 대규모 언어 모델이다.
- 임베딩은 의미 표현, 어텐션은 문맥 선택이다.
- 컨텍스트 윈도우는 기억 용량처럼 보이지만, 정확히는 한 번에 참조 가능한 입력 범위다.
- 챗봇 제품은 모델 하나가 아니라 시스템 전체 결과물이다.

## 자주 하는 오해
- LLM은 사실을 저장한 데이터베이스가 아니다.
- 컨텍스트 윈도우가 길다고 항상 더 정확한 것은 아니다.
- 파인튜닝만 하면 최신 지식을 자동으로 아는 것은 아니다.
- "사람처럼 대화한다"는 것이 "이해 방식이 사람과 같다"는 뜻은 아니다.

## 체크포인트
- 토큰과 임베딩의 차이를 설명할 수 있는가?
- 어텐션이 왜 트랜스포머의 핵심인지 말할 수 있는가?
- 컨텍스트 윈도우와 추론 비용의 관계를 이해했는가?
- 파인튜닝이 모델 구조 자체를 바꾸는 일이 아니라는 점을 아는가?
- 챗봇과 LLM을 분리해서 설명할 수 있는가?

## 한 줄 요약
LLM은 트랜스포머를 기반으로 토큰의 문맥 관계를 학습해 다음 토큰을 예측하는 모델이며, 실제 제품 품질은 그 위에 얹는 시스템 설계까지 함께 봐야 한다.
