# 파인튜닝 vs RAG

## 이 문서에서 배우는 것
- 파인튜닝과 RAG가 무엇이 다른지 명확히 구분한다.
- 각각 어떤 문제에 적합한지 이해한다.
- 실무에서 둘을 어떻게 함께 쓸 수 있는지 감을 잡는다.
- 기술 선택을 할 때 어떤 질문을 먼저 해야 하는지 정리한다.

## 왜 중요한가
LLM 프로젝트를 시작하면 자주 듣는 질문이 "이건 파인튜닝해야 하나요, RAG 붙여야 하나요?"이다. 둘은 목적이 다르기 때문에, 차이를 잘못 이해하면 비용과 시간이 크게 낭비된다.

많은 경우 문제는 "모델이 무엇을 알아야 하느냐"보다 "모델이 어떤 방식으로 답해야 하느냐"와 "어떤 정보를 참고해야 하느냐"로 나뉜다. 파인튜닝과 RAG는 이 두 축에 서로 다르게 대응한다.

## 핵심 개념 설명
파인튜닝은 모델의 파라미터를 추가 학습으로 조정해 특정 스타일, 형식, 도메인 행동에 더 잘 맞게 만드는 방식이다. RAG는 모델 파라미터를 바꾸지 않고, 외부 문서를 검색해 컨텍스트로 넣어 주는 방식이다.

핵심 비교:

| 항목 | 파인튜닝 | RAG |
| --- | --- | --- |
| 무엇을 바꾸나 | 모델 내부 파라미터 | 입력 컨텍스트 |
| 잘하는 일 | 말투, 형식, 작업 습관 적응 | 최신 정보, 사내 문서 반영 |
| 지식 업데이트 | 다시 학습 필요 | 문서 갱신으로 반영 가능 |
| 비용 구조 | 학습 비용 발생 | 검색 인프라 비용 발생 |
| 대표 리스크 | 데이터 품질에 따른 과적합 | 검색 실패, 문맥 품질 저하 |

## 질문으로 구분하기
아래 질문을 순서대로 던져 보면 구분이 쉬워진다.

### 질문 1: 최신 정보가 중요한가
중요하다면 RAG 쪽 가능성이 크다. 예를 들어 사내 정책, 상품 재고, 가격표, 법규, 공지사항은 계속 바뀐다. 이런 정보는 모델 내부에 넣어 두기보다 검색해서 쓰는 편이 낫다.

### 질문 2: 답변 형식이나 행동 패턴이 중요한가
중요하다면 파인튜닝 가능성이 있다. 예를 들어 "항상 특정 JSON 스키마로 응답", "브랜드 톤앤매너 유지", "정해진 절차에 따라 답변" 같은 것은 행동 습관 문제에 가깝다.

### 질문 3: 둘 다 필요한가
많은 실무 프로젝트는 둘 다 필요하다. 예를 들어 상담봇이 항상 정해진 응답 형식을 유지하면서도 최신 상품 정책을 참고해야 한다면, 행동은 파인튜닝 또는 강한 프롬프트로 맞추고, 지식은 RAG로 공급할 수 있다.

## 예시 / 비유
파인튜닝은 직원을 장기 교육해 일하는 방식을 바꾸는 것과 비슷하다. RAG는 일을 할 때마다 최신 매뉴얼을 옆에 두고 참고하게 만드는 것에 가깝다.

더 구체적인 예시:
- `"항상 회의록을 우리 회사 템플릿으로 정리"`: 파인튜닝 또는 프롬프트
- `"최신 사내 규정 문서를 보고 답변"`: RAG
- `"회사 말투 + 최신 규정 반영"`: 둘의 조합

## 의사결정 예시

### 사례 1: 브랜드 고객센터 챗봇
- 브랜드 말투 일관성 필요
- 최신 배송 정책 반영 필요
- 결론: 파인튜닝/프롬프트 + RAG 조합

### 사례 2: 고정된 형식의 문서 변환기
- 최신 지식은 거의 필요 없음
- 출력 형식 안정성이 중요
- 결론: 프롬프트 또는 파인튜닝 중심

### 사례 3: 사내 위키 질의응답
- 정보 최신성 매우 중요
- 근거 출처 제시 필요
- 결론: RAG 중심

## 한눈에 정리
- 파인튜닝: 모델의 "습관"을 바꾼다.
- RAG: 모델의 "참고자료"를 붙인다.
- 둘은 대체재가 아니라 보완재일 수 있다.
- 기술 선택 전에 "행동 문제인지, 지식 문제인지" 먼저 구분해야 한다.

## 자주 하는 오해
- RAG를 쓰면 모델이 그 지식을 영구적으로 배운다고 생각하기 쉽다.
- 파인튜닝을 하면 최신 정보가 자동으로 업데이트된다고 오해하기 쉽다.
- 둘 중 하나만이 정답인 프로젝트는 생각보다 많지 않다.
- 출력 품질 문제를 모두 파인튜닝으로 해결하려 하면 비용이 커질 수 있다.

## 체크포인트
- 파인튜닝과 RAG가 각각 무엇을 바꾸는지 설명할 수 있는가?
- 최신 지식 반영 문제에 왜 RAG가 자주 쓰이는지 말할 수 있는가?
- 형식 일관성과 사실 최신성을 서로 다른 문제로 볼 수 있는가?
- 내가 다루는 문제를 "행동/형식"과 "지식/근거"로 나눠 설명할 수 있는가?

## 한 줄 요약
파인튜닝은 모델 행동을 조정하고, RAG는 외부 지식을 연결하므로, 무엇을 고치려는 문제인지 먼저 구분해야 한다.
