# 고전 딥러닝 아키텍처와 한계

## 이 문서에서 배우는 것
- CNN, RNN, GAN이 각각 어떤 문제를 해결하려고 나왔는지 이해한다.
- 신경망의 표현력과 실제 학습 난이도의 차이를 구분한다.
- 깊은 신경망에서 자주 생기는 고질적 문제와 해결 방향을 정리한다.
- 왜 트랜스포머가 등장했는지도 자연스럽게 연결해 본다.

## 왜 중요한가
요즘은 LLM과 트랜스포머가 중심이라서 CNN, RNN, GAN을 "예전 기술"처럼 느끼기 쉽다. 하지만 이 구조들은 딥러닝이 어떤 문제를 만나며 발전했는지 보여 주는 중요한 이정표다. 이미지, 순서 데이터, 생성 문제를 각각 어떻게 바라봤는지 알면 현대 모델도 더 잘 이해할 수 있다.

또한 Universal Approximation Theorem 같은 이론만 보면 신경망이 모든 걸 할 수 있을 것처럼 느껴질 수 있다. 하지만 실제로는 기울기 소실, 과적합, 긴 의존성 문제, 학습 불안정 같은 현실적인 문제가 계속 따라붙는다.

## 핵심 개념 설명

### 1. CNN: 이미지와 공간 구조를 다루기 위해
CNN은 합성곱 연산을 통해 이미지처럼 공간 구조가 중요한 데이터를 더 효율적으로 처리하도록 나온 구조다.

왜 중요했나:
- 픽셀 간의 지역적 패턴을 잘 잡을 수 있다
- 같은 필터를 여러 위치에 재사용할 수 있어 효율적이다
- 이미지 분류와 인식 문제에서 강력했다

### 2. RNN: 순서를 다루기 위해
RNN은 이전 상태를 다음 계산에 반영하는 구조라서, 문장이나 시계열처럼 순서가 중요한 데이터를 다루기 위해 등장했다.

왜 중요했나:
- "앞에서 본 정보"를 어느 정도 기억하면서 처리할 수 있다
- 초기 자연어처리와 시계열 문제에서 널리 쓰였다

한계도 분명했다.
- 긴 문맥을 기억하기 어렵다
- 기울기 소실과 폭주 문제가 잘 생긴다

이 한계가 나중에 어텐션과 트랜스포머의 중요성을 키웠다.

### 3. GAN: 생성 문제를 경쟁 구조로 풀기 위해
GAN은 생성자와 판별자가 경쟁하면서 학습하는 구조다.
- 생성자: 그럴듯한 데이터를 만든다
- 판별자: 진짜와 가짜를 구분한다

왜 중요했나:
- 이미지 생성의 품질을 크게 끌어올렸다
- 생성 문제를 "대결 구조"로 본 대표적인 아이디어다

한계:
- 학습이 불안정할 수 있다
- 모드 붕괴 같은 문제가 생길 수 있다

### 4. Universal Approximation Theorem의 의미와 한계
이 정리는 신경망이 충분히 크면 복잡한 함수도 근사할 수 있음을 보여 준다. 하지만 현실에서는 다음 질문이 더 중요하다.
- 그 함수를 실제로 학습할 수 있는가?
- 데이터가 충분한가?
- 계산 자원이 충분한가?
- 일반화가 되는가?

즉, 표현 가능성과 학습 가능성은 다르다.

### 5. 깊은 신경망의 고질적 문제

| 문제 | 뜻 | 왜 어려운가 |
| --- | --- | --- |
| 기울기 소실 | 뒤로 갈수록 gradient가 너무 작아짐 | 앞쪽 층이 거의 학습되지 않음 |
| 기울기 폭주 | gradient가 지나치게 커짐 | 학습이 불안정해짐 |
| 과적합 | 훈련 데이터에만 너무 잘 맞음 | 실제 데이터에서 성능 저하 |
| 학습 불안정 | 손실이 흔들리거나 발산 | 재현성과 튜닝이 어려움 |
| 긴 의존성 문제 | 먼 과거 정보를 반영하기 어려움 | RNN 계열에서 특히 심함 |

### 6. 해결 방향

| 해결 방향 | 무엇을 완화하나 | 대표 예시 |
| --- | --- | --- |
| 더 나은 초기화 | 기울기 흐름 개선 | Xavier, He 초기화 |
| 활성화 함수 선택 | 학습 흐름 개선 | ReLU 계열 |
| 정규화 | 학습 안정화 | BatchNorm, LayerNorm |
| 스킵 연결 | 깊은 네트워크 학습 보조 | ResNet |
| 어텐션 | 긴 의존성 문제 완화 | Transformer |
| 데이터 증강 / 정규화 | 과적합 완화 | Augmentation, Dropout |

### 7. 왜 트랜스포머가 중요해졌나
CNN, RNN, GAN 각각은 특정 문제를 잘 해결했지만, 긴 문맥과 병렬 처리, 범용성 측면에서 한계가 있었다. 트랜스포머는 어텐션을 중심으로 이 문제를 상당 부분 해결하면서 NLP와 생성형 AI의 중심 구조가 됐다.

즉, 트랜스포머는 갑자기 하늘에서 떨어진 구조가 아니라, 기존 딥러닝 구조들의 한계를 거치며 등장한 다음 단계라고 볼 수 있다.

## 예시 / 비유
세 구조를 서로 다른 전문가 팀으로 생각해 볼 수 있다.
- CNN은 이미지를 잘 보는 시각 전문가
- RNN은 순서를 따라 읽는 기록 전문가
- GAN은 창작자와 감정자가 서로 경쟁하며 수준을 끌어올리는 창작 팀

하지만 팀이 커질수록 소통이 어려워지고, 의사결정이 흔들리며, 기억이 길게 이어지지 않는 문제가 생긴다. 깊은 신경망의 고질적 문제도 비슷한 구조적 어려움으로 볼 수 있다.

## 한눈에 정리

| 구조/개념 | 핵심 역할 | 대표 한계 |
| --- | --- | --- |
| CNN | 공간 패턴 처리 | 장기 문맥 처리에는 약함 |
| RNN | 순차 데이터 처리 | 긴 의존성, 기울기 문제 |
| GAN | 생성 문제 학습 | 불안정한 학습 |
| UAT | 신경망의 표현 가능성 설명 | 학습 난이도는 해결 못 함 |
| 깊은 신경망 문제 | 실제 학습에서 나타나는 병목 | 초기화, 정규화, 구조 개선 필요 |

## 자주 하는 오해
- CNN, RNN, GAN은 이제 다 쓸모없다고 생각하기 쉽지만, 각 구조가 해결하려던 문제는 여전히 중요하다.
- 신경망이 복잡한 함수를 근사할 수 있다는 말이 곧 실전 성능 보장을 뜻하지는 않는다.
- 깊은 모델은 무조건 좋은 것이 아니라, 학습 안정성 문제를 함께 해결해야 한다.
- 트랜스포머는 기존 구조를 완전히 무의미하게 만든 것이 아니라, 그 한계를 계승하며 발전한 구조다.

## 체크포인트
- CNN, RNN, GAN이 각각 왜 등장했는지 설명할 수 있는가?
- Universal Approximation Theorem이 말하는 것과 말하지 않는 것을 구분할 수 있는가?
- 깊은 신경망에서 기울기 소실과 과적합이 왜 문제인지 말할 수 있는가?
- 트랜스포머를 기존 구조의 한계와 연결해서 설명할 수 있는가?

## 한 줄 요약
CNN, RNN, GAN은 각기 다른 데이터 문제를 해결하기 위해 발전한 구조이며, 그 한계를 이해하면 왜 현대 딥러닝이 트랜스포머로 이어졌는지 보인다.
