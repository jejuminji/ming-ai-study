# AI 학습 로드맵

## 이 문서에서 배우는 것
- AI를 공부할 때 어떤 순서로 큰 그림을 잡아야 하는지 이해한다.
- 머신러닝, 딥러닝, LLM, RAG, 에이전트가 어떻게 이어지는지 본다.
- 지금 내가 어디쯤 있는지 학습 위치를 점검한다.
- "무엇을 먼저 배우고 무엇을 나중에 배우는지" 우선순위를 세울 수 있다.

## 왜 중요한가
AI는 용어가 많고, 서로 비슷해 보이는 개념도 많다. 처음부터 세부 기술만 따라가면 지금 배우는 것이 어디에 속하는지 놓치기 쉽다. 로드맵이 있으면 "이 개념은 모델 이해 단계인가, 시스템 설계 단계인가"를 구분할 수 있어서 학습 효율이 올라간다.

실무에서도 마찬가지다. 예를 들어 챗봇을 만든다고 할 때 모델만 알아서는 부족하다. 문서 검색은 어떻게 붙일지, 답변 품질은 어떻게 평가할지, 비용과 속도는 어떻게 맞출지까지 함께 봐야 한다. 그래서 AI 학습은 단일 기술 암기보다 "지도를 먼저 그리고 세부를 채우는 방식"이 훨씬 강하다.

## 핵심 개념 설명
쉬운 설명으로 말하면 AI는 "기계를 똑똑하게 만드는 큰 분야"다. 그 안에 데이터를 통해 패턴을 배우는 머신러닝이 있고, 머신러닝 안에 신경망 기반 접근인 딥러닝이 있다. LLM은 딥러닝 기반 언어 모델의 한 종류이고, RAG는 LLM에 외부 지식을 연결하는 방식이며, 에이전트는 모델이 도구를 쓰고 여러 단계를 수행하도록 만든 시스템 패턴이다.

핵심 흐름은 보통 다음과 같다.
1. AI 전체 지도 이해
2. ML과 DL의 차이 이해
3. 데이터, 수학, 평가의 기본 언어 이해
4. 신경망과 트랜스포머 구조 이해
5. 프롬프팅, RAG, 파인튜닝, 에이전트 비교
6. 평가와 시스템 설계 이해

즉, "AI = LLM"으로 바로 들어가기보다, LLM이 전체 지도 안에서 어디쯤 있는지 먼저 이해하는 것이 중요하다.

## 예시 / 비유
AI를 도시라고 보면, 머신러닝은 그 도시의 핵심 산업이다. 딥러닝은 그 산업 안의 대형 공장이고, LLM은 언어를 다루는 전문 공장이다. RAG는 공장에 외부 도서관을 연결한 것이고, 에이전트는 공장이 스스로 도구를 꺼내 여러 일을 처리하게 만든 운영 시스템이다.

입문자 관점의 더 현실적인 예시로 보면:
- `스팸 메일 분류`는 머신러닝/딥러닝 문제다.
- `챗봇 답변 생성`은 LLM 문제다.
- `사내 문서 기반 답변`은 RAG 문제다.
- `문서 찾기 -> 요약 -> 메일 작성 -> 전송` 자동화는 에이전트 문제에 가깝다.

## 한눈에 정리

| 단계 | 핵심 질문 | 대표 문서 | 실제로 익히는 것 |
| --- | --- | --- | --- |
| 전체 지도 | AI 안에 무엇이 있나? | `00`, `01` | 포함 관계와 큰 흐름 |
| 기반 이해 | 왜 수학과 데이터가 필요한가? | `02`, `03` | 데이터, 문제 유형, 평가 감각 |
| 모델 이해 | 신경망과 트랜스포머는 어떻게 다른가? | `04`, `05` | 딥러닝, LLM 구조 |
| 응용 이해 | RAG와 Agent는 무엇을 해결하나? | `06`, `07` | 제품에 붙이는 방식 |
| 운영 이해 | 성능을 어떻게 재고 시스템을 어떻게 설계하나? | `08`, `09` | 평가, 모니터링, 비용 |

## 추천 학습 순서
처음 읽을 때는 아래 순서가 자연스럽다.
1. `00`, `01`로 전체 지도 잡기
2. `02`, `03`으로 기반 개념 익히기
3. `04`, `05`로 모델 구조 이해하기
4. `06`, `07`로 LLM 응용 방식 비교하기
5. `08`, `09`로 실무 운영 관점 붙이기

이 순서를 따르는 이유는, 앞 문서가 뒤 문서의 전제 역할을 하기 때문이다. 예를 들어 RAG를 이해하려면 먼저 LLM이 무엇인지 알아야 하고, 평가를 이해하려면 먼저 어떤 문제를 풀고 있는지 알아야 한다.

## 실무에서 보는 흐름
예를 들어 "회사 내부 문서를 답변하는 챗봇"을 만든다고 가정해 보자.
1. 먼저 LLM이 무엇인지 이해한다.
2. 최신 문서를 써야 하므로 RAG가 필요하다는 판단을 한다.
3. 답변 품질을 보려면 검색 정확도와 생성 품질을 함께 평가해야 한다.
4. 실제 서비스로 내보내려면 비용, 지연 시간, 로그, 안전장치도 봐야 한다.

이렇게 보면 로드맵은 공부용 순서이면서 동시에 제품 설계 순서와도 연결된다.

## 자주 하는 오해
- AI를 하나의 기술로 생각하기 쉽지만 실제로는 넓은 분야다.
- LLM을 배우면 AI 전부를 아는 것처럼 느끼기 쉽지만, 그것은 일부다.
- 모델만 좋으면 제품도 좋은 것이 아니다. 평가와 시스템 설계가 함께 필요하다.
- 논문 용어를 많이 안다고 해서 전체 구조를 이해한 것은 아니다.

## 체크포인트
- AI, ML, DL, LLM의 포함 관계를 설명할 수 있는가?
- RAG와 에이전트가 모델 그 자체가 아니라 시스템 패턴이라는 점을 이해했는가?
- 내가 지금 배우려는 개념이 전체 흐름의 어디에 있는지 말할 수 있는가?
- "내가 지금 배우는 문서가 다음 문서와 어떻게 연결되는지" 설명할 수 있는가?

## 한 줄 요약
AI 학습은 개별 기술 암기가 아니라 큰 지도 위에서 위치를 잡고, 그 위에 모델·응용·운영 개념을 차례대로 쌓는 과정이다.
