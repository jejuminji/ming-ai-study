# 신경망 학습과 분류의 핵심

## 이 문서에서 배우는 것
- 인공신경망을 함수 관점에서 이해한다.
- weight, bias, 선형 회귀, 경사하강법, 초기화의 관계를 정리한다.
- MLP, 비선형 활성화 함수, Backpropagation이 왜 필요한지 이해한다.
- 이진 분류, 다중 분류, Sigmoid, Softmax, likelihood, MLE의 흐름을 본다.

## 왜 중요한가
딥러닝을 공부하다 보면 `backpropagation`, `sigmoid`, `MLE`, `SGD` 같은 단어가 한꺼번에 쏟아진다. 그런데 이 용어들은 따로 외우면 금방 흐려진다. 반대로 "신경망은 함수이고, 학습은 그 함수의 파라미터를 조정하는 과정"이라는 큰 그림을 잡으면 각각의 개념이 자연스럽게 연결된다.

실무적으로도 이 문서는 중요하다. 모델을 직접 튜닝하지 않더라도, 손실 함수가 무엇인지, 분류 문제에서 왜 sigmoid나 softmax가 나오는지, 왜 학습이 흔들리는지 정도는 알아야 결과를 해석할 수 있다.

## 핵심 개념 설명

### 1. 인공신경망은 함수다
가장 단순한 인공뉴런은 입력을 받아 가중합을 만들고, 거기에 bias를 더한 뒤, 필요하면 활성화 함수를 통과시키는 구조다.

쉽게 말하면:
- `weight`: 어떤 입력을 얼마나 강하게 볼지
- `bias`: 기준점을 어디로 옮길지

즉, 신경망은 입력을 다른 형태의 출력으로 바꾸는 함수들의 연결망이다.

### 2. 선형 회귀에서 신경망으로
선형 회귀는 가장 기본적인 예측 함수다. 입력과 출력이 직선 관계라고 가정하고 학습한다. 이 단순한 출발점이 중요한 이유는, 신경망도 결국은 "함수의 파라미터를 조정해 데이터에 맞춘다"는 점에서 같은 뿌리를 갖기 때문이다.

차이는 표현력이다.
- 선형 회귀: 단순한 관계를 표현
- 퍼셉트론: 선형 분리 문제를 다룸
- MLP: 여러 층과 비선형성을 통해 더 복잡한 관계를 표현

### 3. 왜 비선형 활성화 함수가 필요한가
층을 여러 개 쌓아도 중간에 비선형 활성화 함수가 없으면, 결국 전체는 하나의 큰 선형 함수와 크게 다르지 않다. 그래서 복잡한 곡선 관계를 배우기 어렵다.

대표 활성화 함수:

| 함수 | 어디서 자주 보나 | 핵심 포인트 |
| --- | --- | --- |
| Sigmoid | 이진 분류 출력층 | 0~1 확률처럼 해석하기 쉬움 |
| Tanh | 초기 신경망 설명 | 출력 중심이 0 근처 |
| ReLU | 은닉층 | 간단하고 학습이 잘 되는 편 |
| Softmax | 다중 분류 출력층 | 여러 클래스 확률 분포 생성 |

### 4. 경사하강법과 초기화
학습은 손실 함수를 줄이는 방향으로 파라미터를 움직이는 과정이다. 이때 Gradient Descent, SGD, Adam 같은 최적화 방식이 등장한다.

초기화도 매우 중요하다. 시작점이 너무 나쁘면 학습이 느리거나 불안정해질 수 있다. 그래서 가중치 초기화는 단순한 세팅이 아니라 학습 가능성을 좌우하는 출발선에 가깝다.

### 5. Backpropagation은 왜 필요한가
딥러닝은 층이 많기 때문에, 최종 오류가 각 층의 weight에 얼마나 책임이 있는지 계산해야 한다. 그때 사용하는 것이 Backpropagation이다.

쉬운 설명으로는 "최종적으로 틀린 이유를 뒤에서부터 거슬러 올라가며 각 연결에 나눠 주는 과정"이다.

행렬 미분이 자주 등장하는 이유도 여기에 있다. 실제 신경망 연산은 벡터와 행렬 단위로 이뤄지기 때문에, 역전파를 제대로 이해하려면 행렬 관점이 필요해진다.

### 6. 이진 분류와 다중 분류
분류 문제도 종류가 다르다.

| 문제 | 출력 형태 | 자주 쓰는 함수 |
| --- | --- | --- |
| 이진 분류 | 둘 중 하나 선택 | Sigmoid |
| 다중 분류 | 여러 클래스 중 하나 선택 | Softmax |

퍼셉트론은 분류의 출발점으로 이해하기 좋다. 하지만 딥러닝에서는 단순 퍼셉트론보다, 확률적 해석이 가능한 sigmoid/softmax와 손실 함수 조합이 더 중요하게 등장한다.

### 7. MSE vs likelihood, 그리고 MLE
입문자가 자주 헷갈리는 지점이 바로 여기다. "왜 분류에서 MSE보다 log-likelihood나 cross-entropy를 더 자주 쓰지?"라는 질문이다.

핵심은 문제 해석이다.
- MSE는 오차의 크기를 직접 줄이는 관점에 가깝다.
- likelihood는 "이 데이터가 지금 모델에서 얼마나 그럴듯한가"를 보는 관점에 가깝다.

MLE(Maximum Likelihood Estimation)는 데이터를 가장 그럴듯하게 설명하는 파라미터를 찾는 원리다. 신경망 분류에서도 이 관점이 자주 쓰이고, cross-entropy 손실과 자연스럽게 연결된다.

### 8. Universal Approximation Theorem과 현실의 한계
이 정리는 충분한 크기의 신경망이 매우 다양한 함수를 근사할 수 있다는 아이디어를 준다. 하지만 중요한 단서가 있다.

- 근사할 수 있다는 것과
- 쉽게 학습할 수 있다는 것은 다르다

즉, 이 정리는 신경망의 가능성을 보여 주지만, 실제 학습의 어려움까지 해결해 주지는 않는다.

## 예시 / 비유
신경망 학습을 회사의 피드백 시스템으로 생각해 볼 수 있다.
- weight는 각 팀원이 의사결정에 미치는 영향력
- bias는 기본적인 판단 기준
- 손실 함수는 결과가 얼마나 빗나갔는지
- 경사하강법은 더 나은 방향으로 조직을 조금씩 조정하는 과정
- 역전파는 최종 실패 원인을 각 단계에 나눠서 책임을 추적하는 과정

## 한눈에 정리

| 주제 | 핵심 메시지 |
| --- | --- |
| weight / bias | 입력을 어떤 기준으로 얼마나 볼지 조정한다 |
| 선형 회귀 | 가장 기본적인 예측 함수 출발점이다 |
| MLP | 여러 층과 비선형성으로 표현력을 높인다 |
| 활성화 함수 | 선형 모델 한계를 넘게 해 준다 |
| Backpropagation | 오류를 뒤로 전파해 학습 가능하게 만든다 |
| Sigmoid / Softmax | 분류 문제의 출력 해석에 중요하다 |
| likelihood / MLE | 분류 손실을 확률 관점으로 이해하게 해 준다 |
| UAT | 신경망의 가능성을 보여 주지만 학습 난이도까지 해결하진 않는다 |

## 자주 하는 오해
- weight와 bias는 단순 숫자라서 중요하지 않다고 느끼기 쉽지만, 학습의 핵심 파라미터다.
- 층만 많이 쌓으면 복잡한 문제를 자동으로 풀 수 있는 것은 아니다.
- Universal Approximation Theorem이 있다고 해서 아무 모델이나 잘 학습되는 것은 아니다.
- 분류 문제에서 sigmoid를 쓴다고 끝나는 것이 아니라, 손실 함수와 함께 봐야 한다.
- MLE는 어려운 수학 용어처럼 보이지만, 실제로는 "데이터를 가장 그럴듯하게 설명하는 파라미터 찾기"라는 꽤 실용적인 아이디어다.

## 한 줄 요약
신경망 학습은 함수의 파라미터를 조정해 데이터를 더 잘 설명하는 과정이며, weight·활성화·역전파·분류·MLE가 그 흐름을 이룬다.
