# 머신러닝 기초

## 이 문서에서 배우는 것
- 머신러닝이 무엇을 해결하는 방법인지 이해한다.
- 지도학습, 비지도학습, 자기지도학습, 강화학습의 차이를 구분한다.
- 분류, 회귀, 군집화처럼 문제 유형이 어떻게 나뉘는지 익힌다.
- 데이터, 특징, 모델, 학습, 평가의 기본 흐름을 본다.

## 왜 중요한가
딥러닝과 LLM이 주목받지만, 머신러닝의 기본 구조를 이해하지 않으면 모델을 학습한다는 말 자체가 추상적으로 남는다. 실무에서도 데이터 준비, 문제 정의, 성능 평가의 틀은 여전히 머신러닝 사고방식 위에 있다.

또 하나 중요한 점은, 머신러닝이 단순히 "정답 맞히기"만 하는 것이 아니라는 사실이다. 어떤 문제는 분류이고, 어떤 문제는 숫자 예측이며, 어떤 문제는 구조 발견이고, 어떤 문제는 보상을 통해 행동을 배우는 일이다. 이 구분이 잡혀야 이후의 딥러닝, LLM, 강화학습 문서도 자연스럽게 읽힌다.

## 핵심 개념 설명
머신러닝은 데이터를 통해 패턴을 배우는 방식이다. 사람이 규칙을 모두 직접 쓰지 않고, 입력과 결과의 관계를 모델이 학습하게 만든다.

가장 기본적인 문제 유형은 다음처럼 볼 수 있다.

| 문제 유형 | 무엇을 하나 | 대표 예시 |
| --- | --- | --- |
| 분류 | 범주를 맞힌다 | 스팸/정상 메일 구분 |
| 회귀 | 숫자를 예측한다 | 집값 예측 |
| 군집화 | 비슷한 것끼리 묶는다 | 고객 세그먼트 분리 |
| 표현 학습 | 유용한 특징을 만든다 | 임베딩, 자기지도학습 |
| 정책 학습 | 행동을 배운다 | 게임 AI, 로봇 |

학습 방식도 함께 구분해야 한다.

| 구분 | 입력 | 목표 | 예시 |
| --- | --- | --- | --- |
| 지도학습 | 데이터 + 정답 | 예측 | 분류, 회귀 |
| 비지도학습 | 데이터만 | 구조 발견 | 군집화, 차원 축소 |
| 자기지도학습 | 데이터 내부 신호 | 표현 학습 | 마스킹 예측, 다음 토큰 예측 |
| 강화학습 | 상태, 행동, 보상 | 정책 개선 | 게임, 로봇 |

자기지도학습은 특히 현대 AI에서 중요하다. 정답을 사람이 일일이 붙이지 않아도, 데이터 자체에서 학습 신호를 만들어 표현을 배울 수 있기 때문이다. LLM의 다음 토큰 예측도 넓게 보면 자기지도학습 계열로 이해할 수 있다.

기본 흐름은 보통 다음과 같다.
1. 문제 정의
2. 데이터 수집과 정제
3. 특징 또는 표현 준비
4. 모델 학습
5. 평가와 개선

딥러닝과의 차이도 중요하다. 전통적 머신러닝은 특징을 사람이 설계하는 경우가 많고, 딥러닝은 특징 표현까지 모델이 함께 학습하는 경향이 강하다.

## 예시 / 비유
스팸 메일 문제를 생각해 보자.
- 지도학습에서는 "이 메일은 스팸이다/아니다"라는 정답이 붙어 있다.
- 비지도학습에서는 메일들을 비슷한 것끼리 묶어 본다.
- 자기지도학습에서는 일부 단어나 문장을 가리고 맞히게 해서 표현을 학습한다.
- 강화학습에서는 어떤 행동이 장기적으로 더 좋은 결과를 내는지 보상을 통해 배운다.

## 한눈에 정리
- 머신러닝은 하나의 기술이 아니라 여러 문제 설정의 묶음이다.
- 분류와 회귀는 지도학습에 자주 등장한다.
- 자기지도학습은 현대 LLM과 표현 학습의 중요한 기반이다.
- 강화학습은 정답 라벨 대신 보상을 사용한다.

## 자주 하는 오해
- 머신러닝은 곧 분류 모델이라고 생각하기 쉽지만 범위가 더 넓다.
- 비지도학습과 자기지도학습을 같은 말로 쓰기 쉽지만, 목적과 학습 신호가 다르다.
- 데이터만 많으면 성능이 저절로 좋아지는 것은 아니다.
- 학습 정확도가 높다고 실제 서비스 성능이 좋은 것은 아니다.

## 체크포인트
- 지도학습, 비지도학습, 자기지도학습, 강화학습의 차이를 설명할 수 있는가?
- 분류와 회귀의 차이를 예시와 함께 말할 수 있는가?
- 왜 LLM을 자기지도학습과 연결해서 볼 수 있는지 설명할 수 있는가?
- 전통적 머신러닝과 딥러닝의 차이를 말할 수 있는가?

## 한 줄 요약
머신러닝은 데이터로 패턴이나 행동을 배우는 접근이며, 문제 유형과 학습 방식의 차이를 함께 봐야 전체 흐름이 잡힌다.
