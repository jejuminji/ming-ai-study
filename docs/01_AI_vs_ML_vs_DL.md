# AI vs ML vs DL

## 이 문서에서 배우는 것
- AI, 머신러닝, 딥러닝의 차이를 구분한다.
- 세 개념이 포함 관계라는 점을 이해한다.
- LLM이 어디에 속하는지도 연결해서 본다.

## 왜 중요한가
입문자가 가장 먼저 헷갈리는 부분이 바로 이 구분이다. 실무에서도 "AI를 쓴다"는 말은 너무 넓고, 실제로는 어떤 접근을 쓰는지 구체적으로 말해야 한다. 이 차이를 알아야 문서를 읽을 때도 맥락이 보인다.

## 핵심 개념 설명
AI는 가장 큰 개념이다. 사람이 하던 지능적 작업을 기계가 하도록 만드는 전체 분야를 말한다. 머신러닝은 규칙을 사람이 직접 쓰기보다 데이터를 보고 패턴을 학습하는 접근이다. 딥러닝은 머신러닝 중에서도 여러 층의 신경망을 사용해 복잡한 패턴을 학습하는 방식이다.

정리하면 `AI > ML > DL` 구조다. 그리고 LLM은 딥러닝 기반의 대규모 언어 모델이므로 DL 아래에 놓인다.

비슷한 개념 차이:

| 개념 | 질문 | 핵심 |
| --- | --- | --- |
| AI | 기계를 어떻게 똑똑하게 만들까? | 전체 분야 |
| ML | 데이터를 통해 어떻게 배우게 할까? | 학습 방법 |
| DL | 복잡한 패턴을 어떻게 더 잘 배우게 할까? | 신경망 기반 방법 |

자주 헷갈리는 포인트는 "딥러닝이 곧 AI"라고 생각하는 것이다. 딥러닝은 AI의 중요한 하위 분야지만 전부는 아니다.

## 예시 / 비유
AI를 요리 전체라고 보면, 머신러닝은 요리법을 사람이 일일이 쓰지 않고 재료 데이터를 보고 감을 익히는 방식이다. 딥러닝은 많은 재료와 복잡한 조리 과정을 다룰 수 있는 고급 조리 시스템에 가깝다.

## 한눈에 정리
- AI: 큰 우산
- ML: 데이터로 학습하는 방법
- DL: 신경망을 활용한 ML의 한 종류
- LLM: DL 기반 언어 모델

## 자주 하는 오해
- AI와 ML을 같은 말로 쓰는 경우가 많다.
- 머신러닝은 옛날 기술, 딥러닝은 최신 기술이라고 단순화하면 안 된다.
- 딥러닝이 항상 더 좋은 것은 아니다. 데이터, 비용, 문제 유형에 따라 다르다.

## 체크포인트
- AI, ML, DL의 포함 관계를 말할 수 있는가?
- 딥러닝이 머신러닝의 한 종류라는 점을 설명할 수 있는가?
- LLM이 왜 딥러닝 아래에 들어가는지 설명할 수 있는가?

## 한 줄 요약
AI는 큰 분야이고, ML은 그 안의 학습 방식이며, DL은 ML 안의 신경망 기반 접근이다.
