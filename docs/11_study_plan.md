# 4주 AI 학습 계획

## 이 문서에서 배우는 것
- 이 레포를 기준으로 4주 학습 루트를 세운다.
- 최근 보강한 신경망 기초 커리큘럼을 어디에 끼워 넣을지 정리한다.
- 매주 무엇을 읽고 무엇을 점검할지 구조적으로 본다.
- 문서를 "읽기만 하지 않고 내 것으로 만드는 방식"을 함께 익힌다.

## 왜 중요한가
좋은 자료가 많아도 계획이 없으면 흐름이 끊기기 쉽다. 특히 입문 단계에서는 욕심내서 너무 많은 주제를 한꺼번에 보기보다, 연결되는 순서대로 학습하는 것이 중요하다. 이번 보강 문서들은 경사하강법, 분류, 역전파, CNN/RNN/GAN 같은 전통 딥러닝 기초를 더 세밀하게 다루기 때문에, 학습 계획 안에서 자리를 잡아 주는 것이 중요하다.

이 계획의 목표는 "짧은 시간 안에 다 외우기"가 아니라, 큰 지도를 만들고 자주 헷갈리는 개념들을 연결하는 데 있다. 그래서 매주 읽을 문서뿐 아니라, 어떤 식으로 복습할지도 같이 넣는 것이 좋다.

## 핵심 개념 설명
이 학습 계획의 목표는 빠르게 전문가가 되는 것이 아니라, 흔들리지 않는 개념 지도를 만드는 것이다. 각 주차는 "읽기 + 스스로 설명하기 + 짧은 정리"를 묶어서 진행한다.

이번 계획은 두 층으로 생각하면 좋다.
- 핵심 축: AI 전체 지도, ML, DL, LLM, RAG, 평가, 시스템
- 보강 축: weight/bias, GD/SGD, MLP, 역전파, 분류, MLE, CNN/RNN/GAN

즉, 앞 절반은 "무엇이 무엇인지 구분하는 단계"이고, 뒤 절반은 "왜 그렇게 작동하는지 이해하는 단계"라고 보면 된다.

## 예시 / 비유
헬스장에서 처음부터 모든 기구를 다 배우기보다, 큰 동작 패턴부터 익히는 것과 비슷하다. AI도 먼저 전체 구조를 잡고, 그다음 신경망과 학습 알고리즘의 내부 동작을 보강해야 세부 기술이 제자리를 찾는다.

공부법 비유로 보면:
- `전체 지도 문서`는 과목의 목차
- `기초 수학/ML 문서`는 공통 언어
- `딥러닝/LLM 문서`는 핵심 원리
- `RAG/평가/시스템 문서`는 실무 응용 파트

## 주차별 실행 가이드

### 1주차
- AI 전체 지도와 핵심 용어를 익힌다.
- `AI`, `ML`, `DL` 관계를 스스로 말로 설명해 본다.
- 지도학습, 비지도학습, 자기지도학습, 강화학습의 차이를 정리한다.
- 수학 문서는 공식을 외우지 말고 역할 중심으로 읽는다.

이번 주의 목표는 "낯선 단어를 덜 낯설게 만드는 것"이다. 처음부터 세부 공식에 집착하지 말고, 각 용어가 무슨 역할을 하는지만 잡아도 충분하다.

### 2주차
- 인공신경망을 "함수" 관점으로 이해한다.
- weight, bias, 선형 회귀, 경사하강법, GD vs SGD를 연결한다.
- `Adam`, `AdamW`, `SGD`가 왜 서로 다른 최적화기인지 정리한다.
- MLP와 비선형 활성화 함수가 왜 필요한지 말로 설명해 본다.

이번 주는 "모델이 어떻게 배우는지"를 이해하는 구간이다. 구현 세부보다도, 손실이 줄어드는 방향으로 파라미터를 갱신한다는 흐름을 잡는 것이 핵심이다.

### 3주차
- Backpropagation이 왜 깊은 신경망 학습의 핵심인지 이해한다.
- 이진 분류, 다중 분류, Sigmoid, Softmax, likelihood, MLE 흐름을 정리한다.
- Universal Approximation Theorem이 의미하는 것과 한계를 함께 이해한다.
- CNN, RNN, GAN의 역할 차이와 깊은 신경망의 고질적 문제를 정리한다.

이번 주는 "딥러닝 내부 문법"을 정리하는 시간이다. 이름을 외우는 것보다 왜 이런 구조가 나왔는지를 이해하면 훨씬 오래 기억된다.

### 4주차
- 트랜스포머와 LLM의 기본 구조를 이해한다.
- 프롬프팅, RAG, 파인튜닝, 에이전트의 차이를 비교표 중심으로 복습한다.
- 어떤 지표를 왜 쓰는지 정리한다.
- AI 시스템을 모델 하나가 아니라 구성 요소 묶음으로 바라본다.

마지막 주는 "이제 이걸 실제 제품으로 연결하면 어떻게 되나"를 보는 단계다. 여기까지 오면 단순 용어 암기가 아니라 설계 대화를 이해할 수 있게 된다.

## 추천 공부 루틴
하루 30분에서 60분 정도라면 아래 방식이 무난하다.
1. 문서 읽기 20~30분
2. 핵심 개념 3줄 요약 5분
3. "이걸 남에게 설명한다면?" 방식으로 말해 보기 5분
4. 헷갈린 용어만 따로 메모 5분

읽기만 하면 이해한 것처럼 느끼기 쉽다. 짧게라도 말로 설명하거나 글로 정리하면 이해가 훨씬 단단해진다.

## 주차별 산출물 예시
- 1주차: `AI > ML > DL > LLM` 포함 관계 그림
- 2주차: `입력 -> 예측 -> 손실 -> 역전파 -> 파라미터 업데이트` 흐름도
- 3주차: `MLP / CNN / RNN / GAN` 비교표
- 4주차: `사용자 입력 -> 검색 -> 모델 -> 평가` 구조도

이런 산출물이 있으면 "읽은 것"이 아니라 "내가 이해한 것"으로 바뀐다.

## 체크리스트
- 매주 문서 2~4개만 읽고 핵심을 짧게 정리했는가?
- 각 문서의 체크포인트 질문에 답해 보았는가?
- "정의 암기"보다 "차이 설명"에 집중했는가?
- 신경망 기초와 LLM 문서를 따로 보지 않고 연결해서 이해하려 했는가?
- 마지막 주에 전체 흐름을 한 장으로 정리해 보았는가?

## 자주 하는 오해
- 짧은 기간에 모든 세부 기술을 다 이해하려고 하면 오히려 흐름을 놓치기 쉽다.
- 읽기만 하고 설명하지 않으면 이해한 것처럼 착각하기 쉽다.
- LLM만 보면 AI를 다 본 것처럼 느끼기 쉽지만, 그 이전의 신경망 기초가 매우 중요하다.
- CNN, RNN, GAN 같은 구조는 옛 개념이라서 안 봐도 된다고 생각하기 쉽지만, 딥러닝 사고방식을 잡는 데 도움이 된다.
