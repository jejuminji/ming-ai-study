# 내가 진행한 AI 프로젝트 정리

## 이 문서의 목적
- 지금까지 진행한 AI 프로젝트를 한눈에 정리한다.
- 어떤 기술을 직접 다뤘는지 흐름으로 연결해 본다.
- 이후 포트폴리오, 자기소개서, 학습 회고 문서의 기반으로 활용한다.

| --- | --- | --- |

| RAG 기반 NPC 챗봇 구축 | RAG, 검색, 캐릭터 응답 | 문서 기반 지식 주입형 NPC 챗봇 구현 |

| SLM + TTS + RAG 구현 | 음성합성, 경량모델, 멀티모달 흐름 | 응답 생성부터 음성 출력까지 연결 |

| 강화학습 기반 몬스터 AI 구축 | RL, 게임 AI | 보상 기반 행동 학습 구조 실험 |
| 일본어 감정분석기 로컬 제작 | NLP, 분류, 일본어 처리 | 로컬 환경에서 일본어 감정분석 모델 구현 |

## 프로젝트별 정리

### 1. Stable Diffusion XL용 LoRA 로컬 제작
- 이미지 생성 모델을 직접 다루기 위해 로컬 환경에서 SDXL용 LoRA를 제작했다.
- 대규모 전체 모델을 다시 학습하기보다, LoRA 방식으로 특정 스타일이나 표현을 효율적으로 반영하는 흐름을 경험했다.
- 이 프로젝트를 통해 모델 학습 비용, 데이터 준비, 체크포인트 관리, 결과 비교의 중요성을 체감했다.

### 2. LLM 파인튜닝 / LoRA 학습
- LLM을 목적에 맞게 조정하기 위해 파인튜닝과 LoRA 기반 학습을 진행했다.
- 특히 경량 모델 또는 SLM 구축에 도움이 되는 방향으로, 전체 모델 재학습보다 현실적인 튜닝 전략을 다뤘다.
- 이 경험으로 "모델 자체를 바꾸는 것"과 "프롬프트나 RAG로 보완하는 것"의 차이를 더 분명히 이해하게 됐다.

### 3. RAG 기반 NPC 챗봇 구축
- 캐릭터형 대화 시스템을 만들기 위해 RAG 기반 NPC 챗봇을 구축했다.
- 단순 생성형 응답이 아니라, 필요한 배경 설정이나 세계관 문서를 검색해 답변에 반영하는 구조를 다뤘다.
- 이 과정에서 검색 품질이 캐릭터 일관성과 답변 신뢰도에 직접 영향을 준다는 점을 경험했다.

### 4. 소설 축약형 RAG 문서 로컬 제작
- 긴 소설이나 서사형 텍스트를 그대로 넣기보다, 검색과 참조에 적합한 형태로 축약형 문서를 로컬에서 제작했다.
- 문서 분할, 요약, 구조화가 RAG 품질에 얼마나 중요한지 확인할 수 있었다.
- 이 경험은 "RAG 성능은 모델만이 아니라 문서 준비 단계에서 크게 갈린다"는 점을 보여 준다.

### 5. SLM + TTS + RAG 구현
- 경량 언어 모델(SLM), 텍스트 음성 변환(TTS), RAG를 연결한 시스템을 구현했다.
- 단순 텍스트 응답을 넘어서, 실제 인터랙션 흐름 안에서 생성 결과를 음성으로 전달하는 경험을 쌓았다.
- 이 프로젝트는 모델 단일 성능보다 시스템 통합 능력이 중요하다는 점을 잘 보여 준다.

### 6. 데이터셋 생성 API
- 학습용 데이터를 더 효율적으로 만들기 위해 데이터셋 생성 API를 구현했다.
- 반복적인 수작업 대신, 데이터 생성 흐름을 API 형태로 다루며 자동화 관점을 익혔다.
- 이 경험은 학습 이전에 데이터 파이프라인이 얼마나 중요한지 체감하게 해 준 작업이었다.

### 7. 프롬프트 최적화
- 모델 성능을 바꾸지 않고도 출력 품질을 개선하기 위해 프롬프트를 최적화했다.
- 지시 방식, 출력 형식, 예시 제공 방식 등을 조정하며 응답 안정성과 일관성을 높이는 방향을 실험했다.
- 이를 통해 "좋은 결과는 모델 크기만이 아니라 입력 설계에서도 나온다"는 점을 실무적으로 익혔다.

### 8. 강화학습 통한 몬스터 AI 구축
- 게임형 환경에서 몬스터 AI를 강화학습 기반으로 구축했다.
- 규칙을 직접 하드코딩하는 대신, 보상을 통해 행동 전략을 학습시키는 접근을 다뤘다.
- 이 프로젝트는 생성형 AI와는 다른 축의 AI 학습 방식도 직접 경험했다는 점에서 의미가 있다.

### 9. 일본어 감정분석기 로컬 제작
- 일본어 텍스트를 대상으로 감정분석기를 로컬 환경에서 제작했다.
- 언어별 전처리, 분류 문제 설정, 데이터 품질의 영향을 실제로 다뤄 볼 수 있었다.
- 이 경험은 LLM뿐 아니라 전통적인 NLP 분류 태스크에도 실전 감각이 있다는 점을 보여 준다.

## 내가 다뤄 본 기술 영역

| 영역 | 관련 경험 |
| --- | --- |
| 생성형 이미지 | SDXL, LoRA |
| 생성형 언어 모델 | LLM 파인튜닝, LoRA, SLM |
| 검색 결합형 시스템 | RAG, 문서 전처리, NPC 챗봇 |
| 음성 결합 시스템 | TTS + SLM + RAG |
| 데이터 엔지니어링 | 데이터셋 생성 API |
| 프롬프트 엔지니어링 | 프롬프트 최적화 |
| 게임 AI | 강화학습 기반 몬스터 AI |
| NLP 분류 | 일본어 감정분석 |

## 이 경험들로 보이는 강점
- 모델 자체를 학습시키는 경험과 시스템으로 묶는 경험을 모두 가지고 있다.
- LLM만 다룬 것이 아니라 이미지 생성, NLP 분류, 게임 AI까지 폭넓게 경험했다.
- 로컬 환경 구축 경험이 있어, 단순 API 사용을 넘어 직접 실험하고 조정하는 흐름에 익숙하다.
- RAG, 파인튜닝, 프롬프트 최적화의 차이를 실제 프로젝트 맥락에서 이해하고 있다.

## 클라우드 / 외부 작업 흔적 메모
- 현재 로컬 레포에서는 개별 클라우드 작업 산출물이나 배포 설정 파일은 확인되지 않았다.
- 다만 Git 원격 저장소 연결 흔적은 확인되며, 현재 레포는 `https://github.com/jejuminji/ming-ai-study.git`와 연결되어 있다.
- 따라서 클라우드에서 별도로 작업한 내용이 있다면, 이 문서에 추후 아래 항목을 추가해 확장할 수 있다.

## 포트폴리오용 짧은 소개 문장
로컬 환경에서 SDXL LoRA 제작, LLM/SLM 파인튜닝, RAG 기반 NPC 챗봇, TTS 결합형 시스템, 데이터셋 생성 API, 강화학습 기반 게임 AI, 일본어 감정분석기까지 직접 구현하며 생성형 AI와 전통적 ML/NLP를 폭넓게 다뤘다.
