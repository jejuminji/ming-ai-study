# AI 학습 알고리즘 정리

## 이 문서에서 배우는 것
- AI에서 자주 나오는 학습 알고리즘과 최적화 기법을 큰 흐름으로 이해한다.
- `GD`, `SGD`, `Adam`, `PPO`, `CQL` 같은 이름이 각각 어디에 쓰이는지 구분한다.
- LLM 학습과 강화학습에서 알고리즘이 어떤 역할을 하는지 연결해서 본다.

## 왜 중요한가
AI를 공부하다 보면 모델 이름보다 알고리즘 이름이 더 많이 등장할 때가 있다. 그런데 `Adam`, `PPO`, `CQL`, `SGD` 같은 단어는 맥락 없이 보면 외우기만 하고 이해가 잘 안 된다. 이 문서는 알고리즘을 공식 중심이 아니라 "무슨 문제를 해결하려고 등장했는가" 중심으로 정리한다.

실무에서도 알고리즘 이름을 안다는 것은 단순 용어 암기가 아니라, 학습 방식과 시스템 제약을 이해한다는 뜻에 가깝다. 예를 들어 LLM 파인튜닝에서는 어떤 최적화기를 쓰는지가 중요하고, 게임 AI나 행동 학습에서는 어떤 강화학습 알고리즘을 쓰는지가 안정성과 성능을 크게 바꾼다.

## 핵심 개념 설명

### 먼저 큰 분류부터 보기
AI에서 자주 보는 알고리즘은 크게 네 묶음으로 볼 수 있다.

| 분류 | 질문 | 대표 예시 |
| --- | --- | --- |
| 최적화 알고리즘 | 가중치를 어떻게 업데이트할까? | GD, SGD, Momentum, Adam, AdamW |
| 지도학습용 학습 방식 | 정답 데이터로 어떻게 학습할까? | Linear Regression, Mini-batch 학습 |
| 강화학습 알고리즘 | 보상을 바탕으로 행동을 어떻게 개선할까? | DQN, PPO |
| 오프라인 강화학습 알고리즘 | 새로 탐험하지 않고 기존 데이터로만 어떻게 배우나? | CQL |

핵심은 이 점이다.
- `GD`, `SGD`, `Adam`은 주로 최적화기 또는 최적화 방식이다.
- `PPO`는 강화학습 알고리즘이다.
- `CQL`은 오프라인 강화학습 알고리즘이다.

즉, 서로 같은 종류의 이름이 아니다. 그래서 같은 선상에서 단순 비교하면 헷갈리기 쉽다.

## 1. 최적화 알고리즘: 가중치를 어떻게 고칠까

### Gradient Descent
Gradient Descent는 손실 함수를 줄이는 방향으로 조금씩 이동하는 가장 기본적인 최적화 아이디어다. 쉬운 설명으로는 "지금 틀린 정도를 보고, 더 덜 틀리는 방향으로 한 걸음 움직이는 것"이라고 생각하면 된다.

#### 어디에 쓰나
- 선형 회귀 같은 기초 모델 설명
- 딥러닝 학습의 기본 직관
- Backpropagation 결과를 실제 업데이트로 연결할 때

#### 한계
- 전체 데이터를 한 번에 다 보는 방식은 느릴 수 있다
- 데이터가 많아질수록 계산 부담이 커진다

### SGD
SGD는 Stochastic Gradient Descent의 약자다. 매번 전체 데이터를 다 보는 대신 일부 샘플이나 미니배치만 보고 업데이트한다.

정확한 설명으로는 경사하강법 계열의 최적화기이며, 손실 함수를 줄이는 방향으로 파라미터를 이동시킨다.

#### 어디에 쓰나
- 머신러닝 기본 학습
- 딥러닝의 기초 학습 루프
- 이론 설명의 출발점

#### 장점
- 더 자주 업데이트할 수 있다
- 큰 데이터셋에서도 현실적으로 학습이 가능하다

#### 한계
- 업데이트가 흔들릴 수 있다
- 학습률 설정에 민감하다

### GD vs SGD

| 항목 | GD | SGD |
| --- | --- | --- |
| 데이터 사용 | 전체 데이터 | 일부 샘플 또는 미니배치 |
| 속도 | 느릴 수 있음 | 더 빠르게 갱신 가능 |
| 안정성 | 비교적 안정적 | 더 흔들릴 수 있음 |
| 실무 활용 | 이론 설명용 | 딥러닝 실전에서 훨씬 자주 사용 |

### Momentum
SGD가 매 순간의 기울기만 보고 움직인다면, Momentum은 이전 방향도 어느 정도 기억하면서 움직인다. 그래서 울퉁불퉁한 경사면에서 덜 흔들리고 더 빠르게 갈 수 있다.

#### 왜 필요했나
- SGD의 느린 수렴과 진동 문제를 줄이기 위해

### Adam
Adam은 딥러닝과 LLM 학습에서 가장 자주 보는 최적화기 중 하나다. 쉬운 설명으로는 "각 파라미터마다 학습 속도를 조금 다르게, 더 똑똑하게 조절해 주는 업데이트 방식"이라고 볼 수 있다.

정확히는 1차 모멘트와 2차 모멘트를 함께 사용해 파라미터별 적응형 학습률을 적용하는 최적화기다.

#### 어디에 쓰나
- 딥러닝 전반
- LLM 사전학습 및 파인튜닝
- LoRA 학습

#### 왜 많이 쓰나
- 초기 설정이 비교적 편하다
- 학습이 안정적인 편이다
- 복잡한 신경망에서도 잘 동작하는 경우가 많다

#### 자주 헷갈리는 포인트
- Adam은 모델 구조가 아니다.
- Adam은 "무엇을 학습할지"가 아니라 "어떻게 업데이트할지"를 정하는 도구다.

### AdamW
AdamW는 Adam에서 가중치 감쇠(weight decay)를 더 잘 분리해 적용한 방식이다. 현대 트랜스포머 학습에서는 Adam보다 AdamW가 자주 언급된다.

#### 왜 중요하나
- 일반화 성능과 정규화 측면에서 더 나은 설정이 필요한 경우가 많기 때문

## 2. LLM에서 자주 연결되는 학습 흐름
LLM에서는 보통 다음 순서로 생각하면 이해가 쉽다.

1. 대규모 텍스트로 사전학습
2. 지도 파인튜닝 또는 instruction tuning
3. 필요하면 선호도 정렬 또는 강화학습 기반 정렬

이 과정에서 `Adam`이나 `AdamW` 같은 최적화기가 자주 쓰인다. 즉 LLM 학습에서 많이 등장하는 알고리즘은 "언어 모델 구조" 자체와 "그 구조를 어떻게 학습시키는가"를 나눠서 봐야 한다.

## 3. 강화학습 알고리즘: 보상으로 행동을 배우기

### 강화학습을 한 문장으로
강화학습은 정답 라벨 대신 보상을 통해 좋은 행동을 배우는 방식이다.

예를 들어 게임 AI는 "이 상황에서 정답 행동이 무엇인지"가 항상 라벨로 주어지지 않는다. 대신 오래 살아남거나 승리하면 보상을 주고, 그 보상을 최대화하도록 정책을 학습한다.

### DQN
DQN은 강화학습 초기에 매우 유명한 알고리즘이다. 상태마다 행동 가치를 추정하는 Q-learning 계열을 딥러닝과 결합한 방식이다.

#### 어디에 쓰나
- 이산 행동 공간
- 게임, 간단한 제어 문제

#### 한계
- 연속 행동 공간에서는 적용이 불편하다
- 안정성 이슈가 있을 수 있다

### PPO
PPO는 현재도 매우 널리 쓰이는 강화학습 알고리즘이다. 쉬운 설명으로는 "정책을 한 번에 너무 크게 바꾸지 않으면서, 조금씩 안정적으로 업데이트하는 방식"이다.

정확히는 policy gradient 계열 알고리즘으로, 이전 정책과 새 정책의 차이가 너무 커지지 않도록 제한하는 아이디어를 사용한다.

#### 어디에 쓰나
- 게임 AI
- 시뮬레이션 제어
- 로봇
- LLM 정렬 단계의 RLHF 계열 설명에서 자주 언급

#### 왜 많이 쓰나
- 비교적 안정적이다
- 구현과 튜닝이 다른 고급 RL 알고리즘보다 덜 까다로운 편이다
- 다양한 환경에서 기준선처럼 자주 사용된다

#### PPO를 이해할 때 중요한 포인트
- PPO는 "행동 정책"을 학습한다.
- 업데이트를 너무 과격하게 하지 않도록 막아 학습을 안정화한다.
- RL에서 자주 보는 대표 알고리즘이지, 모든 상황의 정답은 아니다.

## 4. 오프라인 강화학습: 이미 모인 데이터로만 배우기

### 왜 오프라인 RL이 필요한가
일부 환경에서는 새로 탐험하는 비용이 너무 크다. 예를 들면 실제 서비스 로그, 로봇 제어 기록, 게임 플레이 기록처럼 이미 있는 데이터로만 학습해야 할 때가 있다. 이때 오프라인 강화학습이 필요하다.

### CQL
CQL은 Conservative Q-Learning의 약자다. 쉬운 설명으로는 "데이터에 없는 행동을 모델이 너무 낙관적으로 좋다고 착각하지 않게 조심스럽게 학습하는 방식"이다.

오프라인 RL에서는 직접 탐험하지 못하므로, 모델이 본 적 없는 행동을 과대평가하면 성능이 크게 무너질 수 있다. CQL은 이 문제를 줄이기 위해 더 보수적으로 Q값을 학습한다.

#### 어디에 쓰나
- 오프라인 로그 데이터 기반 정책 학습
- 새 탐험이 위험하거나 비싼 환경

#### 왜 중요한가
- 기존 RL 알고리즘을 그대로 오프라인 데이터에 적용하면 잘 안 되는 이유를 이해하게 해 준다
- "실제로 수집된 행동 범위 안에서 신중하게 배우는 것"의 중요성을 보여 준다

#### 자주 하는 오해
- CQL은 PPO의 대체재가 아니라, 문제 설정이 다른 알고리즘이다.
- PPO는 온라인 상호작용에 더 가깝고, CQL은 오프라인 데이터 학습에 더 가깝다.

## 5. 자주 보이는 알고리즘 이름 빠른 비교

| 이름 | 종류 | 주 사용 맥락 | 핵심 아이디어 |
| --- | --- | --- | --- |
| Gradient Descent | 최적화 방식 | 기초 ML, 선형 회귀 | 전체 손실을 줄이는 방향으로 이동 |
| SGD | 최적화기 | 딥러닝 기초 | 일부 데이터로 자주 업데이트 |
| Momentum | 최적화기 | 딥러닝 기초 | 이전 방향을 반영해 더 부드럽게 이동 |
| Adam | 최적화기 | 딥러닝, LLM | 파라미터별 적응형 업데이트 |
| AdamW | 최적화기 | 트랜스포머, LLM | Adam + 더 나은 weight decay 처리 |
| DQN | 강화학습 | 게임, 이산 행동 | Q값 학습 |
| PPO | 강화학습 | 게임, 제어, RLHF 설명 | 정책을 너무 크게 바꾸지 않음 |
| CQL | 오프라인 RL | 로그 데이터 기반 학습 | 보수적으로 Q값 학습 |

## 6. 비슷한 개념과의 차이 비교

### GD vs SGD
- GD는 전체 데이터를 보고 업데이트하는 기본 개념이다.
- SGD는 일부 데이터만 보고 더 자주 업데이트하는 실전형 방식이다.

### Adam vs PPO
- Adam은 최적화기다.
- PPO는 강화학습 알고리즘이다.
- PPO 안에서도 신경망 파라미터를 업데이트할 때 Adam 같은 최적화기를 함께 쓸 수 있다.

즉, 둘은 경쟁 관계가 아니라 레이어가 다르다.

### PPO vs CQL
- PPO는 보통 환경과 상호작용하며 정책을 개선하는 온라인 강화학습에 가깝다.
- CQL은 이미 수집된 고정 데이터셋으로 학습하는 오프라인 강화학습에 가깝다.

### Adam vs AdamW
- 둘 다 적응형 최적화기다.
- AdamW는 weight decay 처리 방식이 더 분리되어 있어 현대 딥러닝 학습에서 자주 선호된다.

## 예시 / 비유
자동차를 산길에서 운전한다고 생각해 보자.

- `GD`는 지도를 다 펼쳐 놓고 전체 경로를 한 번에 보고 조정하는 방식에 가깝다.
- `SGD`는 당장 보이는 길 상황을 더 자주 반영하며 조금씩 핸들을 돌리는 방식이다.
- `Momentum`은 직전 방향 감각도 반영해 덜 흔들리게 가는 방식이다.
- `Adam`은 바퀴마다 미세하게 다른 조향 보정을 넣어 더 안정적으로 가는 느낌에 가깝다.
- `PPO`는 운전 습관을 한 번에 너무 과격하게 바꾸지 않도록 제한하는 운전법에 가깝다.
- `CQL`은 실제로 가 본 길만 신뢰하고, 가 본 적 없는 위험한 샛길은 너무 좋게 평가하지 않도록 보수적으로 판단하는 방식에 가깝다.

## 한눈에 정리
- 알고리즘 이름은 같은 종류가 아닐 수 있다.
- `GD`, `SGD`, `Adam`은 최적화 쪽, `PPO`는 강화학습, `CQL`은 오프라인 강화학습 쪽이다.
- LLM 학습에서는 주로 최적화기가 자주 보이고, 게임 AI나 정책 학습에서는 RL 알고리즘이 중요하다.
- 어떤 알고리즘이 "더 좋다"보다, 어떤 문제 설정에 맞는지가 더 중요하다.

## 자주 하는 오해
- `Adam`, `PPO`, `CQL`을 같은 종류의 알고리즘으로 비교하면 헷갈린다.
- `GD`와 `SGD`를 이름만 다르고 같은 것으로 생각하기 쉽지만, 데이터 사용 방식과 실전 활용이 다르다.
- PPO를 알면 강화학습 전체를 아는 것은 아니다.
- Adam을 쓴다고 모델이 좋아지는 것이 아니라, 데이터와 목적에 맞는 전체 학습 설정이 중요하다.
- 오프라인 데이터만 있는데 온라인 RL 알고리즘을 그대로 쓰면 잘 안 될 수 있다.

## 체크포인트

- Adam이 모델 구조가 아니라 최적화기라는 점을 설명할 수 있는가?
- PPO와 CQL이 해결하려는 상황 차이를 말할 수 있는가?
- LLM 학습과 강화학습에서 알고리즘이라는 말이 각각 무엇을 가리키는지 구분할 수 있는가?
- PPO 안에서도 Adam 같은 최적화기를 함께 쓸 수 있다는 점을 이해했는가?
