# AI 용어집

## 이 문서에서 배우는 것
- 초보자가 자주 마주치는 AI 용어를 빠르게 확인한다.
- 비슷한 용어를 헷갈리지 않도록 짧고 정확하게 정리한다.
- 다른 문서를 읽다가 막히면 참조하는 사전처럼 활용한다.

## 왜 중요한가
입문 단계에서는 개념 자체보다 용어가 더 어렵게 느껴질 수 있다. 용어를 바로 이해하면 문서를 읽는 속도와 자신감이 함께 올라간다.

## 핵심 개념 설명
이 문서는 가나다순 용어 참고용이다. 깊은 설명보다 "무엇인지, 어디서 쓰는지, 무엇과 헷갈리는지"에 집중한다.

## 용어 정리

| 용어 | 뜻 | 함께 보면 좋은 문서 |
| --- | --- | --- |
| AI | 사람의 지능적 작업을 기계가 수행하게 만드는 큰 분야 | `00`, `01` |
| Agent | 모델이 도구를 쓰고 계획하며 여러 단계를 수행하는 시스템 | `06`, `09` |
| Attention | 입력 요소들 사이에서 무엇을 더 중요하게 볼지 계산하는 메커니즘 | `05` |
| Context Window | 모델이 한 번에 참고할 수 있는 입력 범위 | `05` |
| Deep Learning | 여러 층의 신경망을 사용하는 머신러닝 방식 | `01`, `04` |
| Embedding | 의미를 벡터로 표현한 것 | `05`, `06` |
| Evaluation | 모델이나 시스템 성능을 측정하는 과정 | `08` |
| Fine-tuning | 특정 작업이나 스타일에 맞게 모델을 추가 학습하는 것 | `05`, `07` |
| F1 Score | 정밀도와 재현율의 균형을 보는 지표 | `08` |
| Hallucination | 모델이 그럴듯하지만 틀린 내용을 생성하는 현상 | `06`, `08` |
| Inference | 학습된 모델이 실제 입력에 대해 출력을 만드는 과정 | `05` |
| LLM | 대규모 텍스트 데이터를 학습한 언어 모델 | `05` |
| Machine Learning | 데이터에서 패턴을 학습하는 AI 접근 | `01`, `03` |
| Perplexity | 언어 모델이 다음 토큰을 얼마나 불확실하게 예측하는지 나타내는 지표 | `08` |
| Precision | 양성이라고 예측한 것 중 실제 양성 비율 | `08` |
| Prompt | 모델에게 주는 지시, 맥락, 예시를 포함한 입력 | `06` |
| RAG | 검색으로 외부 문서를 가져와 모델 답변에 반영하는 방식 | `06`, `07` |
| Recall | 실제 양성 중 모델이 찾아낸 비율 | `08` |
| Token | 모델이 텍스트를 처리하는 기본 단위 | `05` |
| Transformer | 어텐션 중심 구조를 가진 딥러닝 아키텍처 | `05` |

## 예시 / 비유
용어집은 여행지의 회화집과 비슷하다. 완전한 문법책은 아니지만, 막히지 않게 해 주는 빠른 번역 도구 역할을 한다.

## 한눈에 정리
- 헷갈리는 용어가 나오면 먼저 이 문서에서 짧게 확인한다.
- 이해가 더 필요하면 연결된 문서로 이동한다.
- 정의보다 문맥과 관계를 함께 본다.

## 자주 하는 오해
- 용어 뜻만 알면 개념도 이해한 것이라고 착각하기 쉽다.
- 비슷한 말이라도 쓰이는 맥락이 다를 수 있다.
- 영어 원어를 보면 어려워 보여도, 대부분은 역할로 이해하면 훨씬 쉽다.

## 체크포인트
- Token, Embedding, Attention의 차이를 짧게 말할 수 있는가?
- RAG와 Fine-tuning을 구분할 수 있는가?
- Agent를 단순 챗봇과 구분해 설명할 수 있는가?

## 한 줄 요약
