# AI 용어집 노션용 표

이 문서는 노션에 붙여넣기 쉽게 만든 AI 용어집입니다.  
기존 학습용 설명 문서보다 더 짧고 구조적으로 정리했고, 실무에서 자주 보이는 최신 용어도 함께 넣었습니다.

## 사용 방법
- 아래 표를 그대로 복사해 노션에 붙여넣어 사용한다.
- 노션에서 표로 바로 안 들어가면, 새 `Simple Table` 또는 데이터베이스 테이블을 만든 뒤 붙여넣는다.
- 이 문서에서는 사용자가 말한 `오픈클로`를 `오픈 웨이트(Open-weight)`와 `클로즈드 모델(Closed model)` 맥락으로 해석했다.

| 기초 | 딥러닝 | DL | 신경망을 깊게 쌓아 학습하는 방식 | 이미지, 음성, 텍스트 성능의 중심 | ML, LLM |
| 기초 | 강화학습 | RL | 보상을 통해 행동을 배우는 방식 | 게임 AI, 제어, 정렬 학습에 중요 | 지도학습 |
| 기초 | 자연어처리 | NLP | 텍스트와 언어를 다루는 AI 분야 | LLM 이전부터 이어진 언어 AI 기반 | LLM |
| 기초 | 컴퓨터 비전 | CV | 이미지와 영상을 이해하는 AI 분야 | 생성형 이미지와 연결됨 | NLP |

| 모델 | 트랜스포머 | Transformer | 어텐션 기반 딥러닝 구조 | 현대 LLM의 기본 구조 | RNN |

| 학습 | 양자화 | Quantization | 모델 숫자 표현을 더 작게 줄이는 방법 | 속도와 메모리 절감에 중요 | Distillation |
| 학습 | 혼합 전문가 | MoE | 일부 전문가 네트워크만 선택적으로 쓰는 구조 | 큰 성능을 더 효율적으로 내기 위해 사용 | Dense Model |
| 학습 | 아담 | Adam | 딥러닝에서 널리 쓰이는 최적화기 | LLM, LoRA 학습에서 자주 등장 | SGD, AdamW |
| 학습 | 아담더블유 | AdamW | Adam의 weight decay 처리를 개선한 최적화기 | 트랜스포머 계열 학습에서 자주 사용 | Adam |

| 정렬/RL | 직접 선호 최적화 | DPO | 선호쌍으로 정책을 직접 조정하는 방식 | RLHF보다 단순하게 정렬할 때 자주 언급 | RLHF |
| 정렬/RL | 근접 정책 최적화 | PPO | 정책을 너무 크게 바꾸지 않게 학습하는 RL 알고리즘 | RL과 RLHF 설명에서 자주 등장 | CQL, Adam |
| 정렬/RL | 보수적 Q 러닝 | CQL | 오프라인 데이터만으로 보수적으로 배우는 RL 알고리즘 | 실제 로그 데이터 기반 학습에 중요 | PPO |
| RAG/검색 | 검색 증강 생성 | RAG | 외부 문서를 검색해 모델 답변에 붙이는 방식 | 최신 정보와 내부 문서 반영에 핵심 | Fine-tuning |
| RAG/검색 | 리트리버 | Retriever | 관련 문서를 먼저 찾아오는 구성 요소 | RAG의 첫 관문 | Reranker |
| RAG/검색 | 리랭커 | Reranker | 찾아온 문서 순서를 다시 정교하게 매기는 모델 | 검색 품질을 높이는 데 도움 | Retriever |
| RAG/검색 | 청킹 | Chunking | 긴 문서를 작은 조각으로 나누는 과정 | 검색 품질과 문맥 주입 품질에 영향 | Summarization |
| RAG/검색 | 벡터 데이터베이스 | Vector DB | 임베딩을 저장하고 유사도 검색하는 저장소 | RAG 인프라 핵심 부품 | 일반 DB |
| RAG/검색 | 그라운딩 | Grounding | 답변이 외부 근거에 기대도록 만드는 것 | 환각을 줄이고 근거 있는 응답에 중요 | Hallucination |
| RAG/검색 | 환각 | Hallucination | 모델이 그럴듯하지만 틀린 내용을 말하는 현상 | 생성형 AI 신뢰성 문제의 핵심 | Grounding |
| 에이전트 | 에이전트 | Agent | 모델이 도구를 쓰고 계획하며 여러 단계를 수행하는 시스템 | 단순 챗봇을 넘어 자동화에 쓰임 | RAG |
| 에이전트 | 도구 사용 | Tool Use | 모델이 검색, 계산, API 등을 호출하는 능력 | 실제 작업 수행력과 직결 | Prompting |
| 에이전트 | 메모리 | Memory | 과거 대화나 상태를 저장해 활용하는 구조 | 장기 작업, 개인화에 중요 | Context Window |
| 에이전트 | 계획 | Planning | 작업을 여러 단계로 나누어 수행 전략을 세우는 것 | 복잡한 업무 자동화의 핵심 | Single-turn |
| 에이전트 | 모델 컨텍스트 프로토콜 | MCP | 모델이 외부 도구와 자원에 표준 방식으로 연결되는 구조 | 에이전트 생태계에서 연결성을 높여 줌 | Tool API |
| 프롬프트 | 프롬프트 엔지니어링 | Prompt Engineering | 더 나은 답을 위해 입력을 설계하는 일 | 모델 성능을 코드 없이 개선할 수 있음 | Fine-tuning |
| 프롬프트 | 컨텍스트 엔지니어링 | Context Engineering | 어떤 정보를 어떤 순서로 모델에 넣을지 설계하는 일 | LLM 앱 품질을 좌우하는 핵심 설계 포인트 | Prompt Engineering |
| 평가 | 평가 | Eval | 모델이나 시스템이 잘 동작하는지 측정하는 일 | 개선과 비교의 기준이 됨 | Benchmark |
| 평가 | 벤치마크 | Benchmark | 여러 모델을 같은 기준으로 비교하는 시험 세트 | 모델 선택과 성능 비교에 필요 | Eval |
| 평가 | 하네스 엔지니어링 | Harness Engineering | 평가를 반복 가능하게 돌리는 테스트 하네스와 실험 체계를 설계하는 일 | 모델 비교, 회귀 테스트, 자동 평가 운영에 중요 | Benchmark |
| 평가 | 가드레일 | Guardrail | 위험한 출력이나 정책 위반을 막는 안전 장치 | 서비스 안전성과 품질 유지에 중요 | Moderation |
| 운영 | 지연 시간 | Latency | 요청 후 응답이 오기까지 걸리는 시간 | 사용자 체감 품질에 직접 영향 | Throughput |
| 운영 | 처리량 | Throughput | 일정 시간 동안 처리할 수 있는 요청량 | 서비스 규모 확장성과 연결 | Latency |
| 운영 | 프롬프트 캐싱 | Prompt Caching | 반복되는 입력을 재사용해 비용과 시간을 줄이는 방식 | LLM 서비스 비용 최적화에 도움 | RAG Cache |
| 운영 | 스펙ulative 디코딩 | Speculative Decoding | 빠른 초안 모델과 큰 모델을 함께 써 추론 속도를 높이는 기법 | 응답 속도 개선에 유용 | 일반 디코딩 |
| 공개성 | 오픈소스 | Open-source | 소스 코드가 공개되고 수정과 배포 조건이 열려 있는 것 | 라이선스와 커뮤니티 활용에 중요 | Open-weight |
| 공개성 | 오픈 웨이트 | Open-weight | 모델 가중치가 공개된 상태 | 직접 실행, 파인튜닝, 분석이 가능해짐 | Open-source, Closed model |
| 공개성 | 클로즈드 모델 | Closed model | 가중치나 내부 구현이 공개되지 않은 모델 | 주로 API 형태로 사용하며 제약이 있을 수 있음 | Open-weight |

## 빠르게 기억할 포인트
- `오픈소스`와 `오픈 웨이트`는 같은 말이 아니다.
- `프롬프트 엔지니어링`은 입력 문장 설계에 가깝고, `컨텍스트 엔지니어링`은 시스템 차원의 정보 배치 설계에 더 가깝다.
- `하네스 엔지니어링`은 단순 테스트 작성이 아니라, 평가를 반복 가능하고 비교 가능하게 만드는 운영 설계에 가깝다.
