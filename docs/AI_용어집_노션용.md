# AI 용어집 노션용 표

이 문서는 노션에 붙여넣기 쉽게 만든 AI 용어집입니다.  
기존 학습용 설명 문서보다 더 짧고 구조적으로 정리했고, 실무에서 자주 보이는 최신 용어도 함께 넣었습니다.

## 사용 방법
- 아래 표를 그대로 복사해 노션에 붙여넣어 사용한다.
- 노션에서 표로 바로 안 들어가면, 새 `Simple Table` 또는 데이터베이스 테이블을 만든 뒤 붙여넣는다.
- 이 문서에서는 사용자가 말한 `오픈클로`를 `오픈 웨이트(Open-weight)`와 `클로즈드 모델(Closed model)` 맥락으로 해석했다.

| 운영 | 모니터링 | Monitoring | 서비스 상태와 품질 변화를 계속 관찰하는 일 | 성능 저하와 장애를 빨리 찾는 데 필요 | Eval |
| 운영 | 프롬프트 캐싱 | Prompt Caching | 반복되는 입력을 재사용해 비용과 시간을 줄이는 방식 | LLM 서비스 비용 최적화에 도움 | RAG Cache |
| 운영 | 스페큘레이티브 디코딩 | Speculative Decoding | 빠른 초안 모델과 큰 모델을 함께 써 추론 속도를 높이는 기법 | 응답 속도 개선에 유용 | 일반 디코딩 |
| 운영 | GGUF | GGUF | 로컬 LLM 실행에서 자주 쓰는 모델 파일 형식 | 양자화 모델 배포와 빠른 로딩에 자주 등장 | SafeTensors, Quantization |
| 공개성 | 오픈소스 | Open-source | 소스 코드가 공개되고 수정과 배포 조건이 열려 있는 것 | 라이선스와 커뮤니티 활용에 중요 | Open-weight |
| 공개성 | 오픈 웨이트 | Open-weight | 모델 가중치가 공개된 상태 | 직접 실행, 파인튜닝, 분석이 가능해짐 | Open-source, Closed model |
| 공개성 | 클로즈드 모델 | Closed model | 가중치나 내부 구현이 공개되지 않은 모델 | 주로 API 형태로 사용하며 제약이 있을 수 있음 | Open-weight |

## 빠르게 기억할 포인트
- `오픈소스`와 `오픈 웨이트`는 같은 말이 아니다.
- `파라미터`는 모델이 배우는 값이고, `하이퍼파라미터`는 사람이 정하는 값이다.
- `프롬프트 엔지니어링`은 입력 문장 설계에 가깝고, `컨텍스트 엔지니어링`은 시스템 차원의 정보 배치 설계에 더 가깝다.
- `정밀도`와 `재현율`은 둘 다 중요하지만, 놓치면 안 되는 문제에서는 보통 `재현율`을 더 민감하게 본다.
- `하네스 엔지니어링`은 단순 테스트 작성이 아니라, 평가를 반복 가능하고 비교 가능하게 만드는 운영 설계에 가깝다.
