# AI 용어집 노션용 표

이 문서는 노션에 붙여넣기 쉽게 만든 AI 용어집입니다.  
기존 학습용 설명 문서보다 더 짧고 구조적으로 정리했고, 실무에서 자주 보이는 최신 용어도 함께 넣었습니다.

## 사용 방법
- 아래 표를 그대로 복사해 노션에 붙여넣어 사용한다.
- 노션에서 표로 바로 안 들어가면, 새 `Simple Table` 또는 데이터베이스 테이블을 만든 뒤 붙여넣는다.
- 이 문서에서는 사용자가 말한 `오픈클로`를 `오픈 웨이트(Open-weight)`와 `클로즈드 모델(Closed model)` 맥락으로 해석했다.

| 인프라 | 에지 추론 | Edge Inference | 클라우드가 아닌 현장 기기에서 직접 추론하는 것 | IoT, 모바일, 로봇 등에서 빠른 응답이 필요할 때 사용 | 온디바이스 |

## 빠르게 기억할 포인트
- `오픈소스`와 `오픈 웨이트`는 같은 말이 아니다.
- `파라미터`는 모델이 배우는 값이고, `하이퍼파라미터`는 사람이 정하는 값이다.
- `프롬프트 엔지니어링`은 입력 문장 설계에 가깝고, `컨텍스트 엔지니어링`은 시스템 차원의 정보 배치 설계에 더 가깝다.
- `정밀도`와 `재현율`은 둘 다 중요하지만, 놓치면 안 되는 문제에서는 보통 `재현율`을 더 민감하게 본다.
- `하네스 엔지니어링`은 단순 테스트 작성이 아니라, 평가를 반복 가능하고 비교 가능하게 만드는 운영 설계에 가깝다.
- `미분`은 한 점에서의 기울기이고, `편미분`은 입력이 여러 개일 때 하나만 움직여 본 기울기다. `그라디언트`는 편미분들을 모아 놓은 벡터다.
- `연쇄 법칙`이 없으면 `역전파`도 없다. 역전파는 연쇄 법칙을 층마다 반복 적용하는 것이다.
- `로그`는 곱셈을 덧셈으로 바꿔 주기 때문에, 확률을 다루는 손실 함수에서 계산 편의와 안정성을 위해 쓴다.
- `정규화`는 문맥에 따라 두 뜻으로 쓰인다. 학습에서는 `Regularization`, 데이터 처리에서는 `Normalization`을 가리킬 수 있다.
- `데이터 정규화`와 `표준화`는 둘 다 값의 스케일을 조정하지만, 기준이 다르다 (최소-최대 vs 평균-분산).
- `GPU`는 병렬 연산, `CPU`는 순차 연산에 강하다. 딥러닝은 행렬 곱이 핵심이라 `GPU`가 필수적이다.
- `VRAM`이 부족하면 모델을 못 올린다. `양자화`와 `LoRA`는 이 한계를 우회하는 실용적 방법이다.
- `CoT`는 프롬프트 기법이고, `추론 모델`은 모델 자체가 생각 과정을 내장한 것이다. 둘 다 정확도를 올리지만 방식이 다르다.
